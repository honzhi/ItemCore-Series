### 3. 核心类实现

#### 3.1 数据模型

- **TrinketSlot** - 饰品槽位定义类
  - `id`: 槽位唯一标识
  - `type`: 允许放置的物品类型（如ring、amulet）
  - `name`: 显示名称
  - `icon`: GUI图标材质
  - `enabled`: 是否启用

- **PlayerTrinketData** - 玩家饰品数据类
  - `playerId`: 玩家UUID
  - `equippedTrinkets`: 已装备饰品映射（槽位ID→物品ID）

#### 3.2 管理器类

- **ConfigManager** - 配置管理器
  - 加载配置文件
  - 提供消息获取方法
  - 配置重载功能

- **SlotLoader** - 槽位加载器
  - 从slots.yml加载槽位定义
  - 支持动态重载

- **TrinketManager** - 饰品管理器（核心）
  - 加载/保存玩家数据
  - 装备/卸下饰品
  - 类型匹配验证

- **DataStorage** - 数据存储
  - YAML文件持久化
  - 玩家数据读写

- **AttributeCalculator** - 属性计算器
  - 计算玩家饰品总属性
  - 应用/移除属性到玩家

#### 3.3 API接口

- **TrinketAPI** - 对外API
  - `getEquippedTrinkets()` - 获取已装备饰品
  - `equipTrinket()` - 装备饰品
  - `unequipTrinket()` - 卸下饰品
  - `getSlot()` - 获取槽位定义

#### 3.4 事件系统

- **TrinketEquipEvent** - 饰品装备事件
- **TrinketUnequipEvent** - 饰品卸下事件

#### 3.5 GUI系统

- **TrinketMenu** - 饰品管理界面
- **GuiListener** - GUI事件监听

#### 3.6 命令系统

- **TrinketCommand** - 主命令处理器
- **TrinketTabCompleter** - 命令补全

## 核心功能实现

### 类型匹配机制

```java
public boolean canEquip(String slotId, CustomItem item) {
    TrinketSlot slot = getSlot(slotId);
    if (slot == null || !slot.isEnabled()) {
        return false;
    }
    // 检查物品类型是否匹配槽位要求的类型
    return item.hasType() && item.getType().equalsIgnoreCase(slot.getType());
}
```

### 属性应用机制

- 玩家加入时加载饰品数据并应用属性
- 装备/卸下饰品时自动更新属性
- 支持生命值、生命恢复、移动速度等属性
