# ItemCoreTrinkets 开发方案

## 1. 项目概述

### 1.1 项目背景

ItemCoreTrinkets 是一款基于 ItemCore 框架开发的饰品插件，为玩家提供额外的装备槽位系统。该插件允许服务器管理员自定义饰品槽位类型，并通过 ItemCore 的物品系统实现属性加成机制。

### 1.2 核心功能点

| 功能点     | 描述                                 | 优先级 |
| ------- | ---------------------------------- | --- |
| 饰品槽位管理  | 用户可自定义槽位类型（如 ring、amulet、bracelet） | 高   |
| 类型匹配机制  | 物品类型与槽位类型匹配才生效属性                   | 高   |
| 属性自动应用  | 识别 ItemCore 物品属性并为玩家附加             | 高   |
| 自定义 GUI | 可视化饰品管理界面                          | 高   |
| 数据持久化   | 保存玩家饰品装备数据                         | 中   |

### 1.3 依赖关系

| 依赖                  | 版本要求   | 类型       |
| ------------------- | ------ | -------- |
| Spigot/Paper        | 1.20+  | 必需       |
| ItemCore            | 1.0.0+ | **硬性依赖** |
| Kyori Adventure API | 4.14+  | 必需       |

**职责**：管理饰品槽位和玩家饰品数据

## 5. API 接口设计

### 5.1 TrinketAPI（对外API）

```java
public class TrinketAPI {
    
    /**
     * 获取玩家已装备的所有饰品
     * @param player 玩家
     * @return 槽位ID -> CustomItem 的映射
     */
    public static Map<String, CustomItem> getEquippedTrinkets(Player player);
    
    /**
     * 装备饰品到指定槽位
     * @param player 玩家
     * @param slotId 槽位ID
     * @param item 物品
     * @return 是否装备成功
     */
    public static boolean equipTrinket(Player player, String slotId, ItemStack item);
    
    /**
     * 卸下指定槽位的饰品
     * @param player 玩家
     * @param slotId 槽位ID
     * @return 卸下的物品，若无则返回null
     */
    public static ItemStack unequipTrinket(Player player, String slotId);
    
    /**
     * 获取槽位定义
     * @param slotId 槽位ID
     * @return 槽位定义，若无则返回null
     */
    public static TrinketSlot getSlot(String slotId);
    
    /**
     * 获取所有槽位
     * @return 所有槽位集合
     */
    public static Collection<TrinketSlot> getAllSlots();
    
    /**
     * 计算玩家饰品总属性
     * @param player 玩家
     * @return 属性容器
     */
    public static AttributeContainer getTrinketAttributes(Player player);
}
```

### 5.2 内部事件

| 事件类名                          | 触发时机    | 用途           |
| ----------------------------- | ------- | ------------ |
| `TrinketEquipEvent`           | 饰品装备成功时 | 允许其他插件监听装备事件 |
| `TrinketUnequipEvent`         | 饰品卸下时   | 允许其他插件监听卸下事件 |
| `TrinketAttributeUpdateEvent` | 属性更新时   | 通知属性变化       |

### 6.3 类型匹配逻辑

```java
// 类型匹配核心逻辑
public boolean canEquip(String slotId, CustomItem item) {
    TrinketSlot slot = getSlot(slotId);
    if (slot == null || !slot.isEnabled()) {
        return false;
    }
    // 检查物品类型是否匹配槽位要求的类型
    return item.hasType() && item.getType().equalsIgnoreCase(slot.getType());
}
```

## 10. ItemCore 物品配置示例

创建 ring 类型饰品的配置示例：

```yaml
# items/trinkets.yml
fire_ring:
  material: DIAMOND
  type: ring
  display-name: "&c烈焰戒指"
  lore:
    - "&7蕴含火焰之力的魔法戒指"
    - "&c攻击伤害 +10"
    - "&c火焰伤害 +5"
  custom-model-data: 5001
  unbreakable: true
  attributes:
    ATTACK_DAMAGE: 10
  element-mastery:
    FIRE: 5
  active-slots:
    - trinkets

ice_amulet:
  material: EMERALD
  type: amulet
  display-name: "&b冰霜护身符"
  lore:
    - "&7蕴含冰霜之力的护身符"
    - "&b魔法抗性 +15"
    - "&b移动速度 +5%"
  custom-model-data: 5002
  unbreakable: true
  attributes:
    MAGIC_RESIST: 15
    MOVEMENT_SPEED: 0.05
  active-slots:
    - trinkets
```

### active-slots 配置说明

`active-slots` 用于指定物品属性生效的槽位：

| 选项          | 说明                    |
| ----------- | --------------------- |
| `main-hand` | 主手槽位生效                |
| `off-hand`  | 副手槽位生效                |
| `head`      | 头盔槽位生效                |
| `chest`     | 胸甲槽位生效                |
| `legs`      | 护腿槽位生效                |
| `feet`      | 靴子槽位生效                |
| `any`       | 任何槽位都生效               |
| `trinkets`  | **仅饰品槽位生效**，在原版槽位中不生效 |

**使用** **`trinkets`** **的效果**：

- 饰品装备在自定义饰品槽位（ring、amulet、bracelet、belt）时，属性正常生效
- 饰品放在背包或原版装备槽位（主手、副手、盔甲）时，属性不生效
- 这样可以防止玩家将饰品当作普通装备使用来获取属性加成
