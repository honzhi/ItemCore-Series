## 🔧 核心实现

### 1. ItemReference 物品引用解析器

支持两种格式的物品引用：

**字符串格式**：
```yaml
output: "itemcore:weapons/iron_sword*1"
materials:
  - "itemcore:misc/iron_ingot*2"
  - "vanilla:STICK*1"
```

**解析逻辑**：
```
"itemcore:weapons/iron_sword*2"
    ↓
source: "itemcore"
category: "weapons"
id: "iron_sword"
amount: 2
```

**核心方法**：
- `fromString(String str)` - 从字符串解析
- `toItemStack()` - 转换为 ItemStack
- `isItemCore()` - 判断是否为 ItemCore 物品
- `isValid()` - 验证引用有效性

### 2. ConditionChecker 条件检查器

**支持的条件类型**：

| 类型 | 说明 | 检查逻辑 |
|------|------|----------|
| `level` | 等级条件 | `player.getLevel() >= value` |
| `permission` | 权限条件 | `player.hasPermission(node)` |
| `money` | 经济条件 | Vault API 检查余额 |

**返回结果**：
```java
CheckResult {
    boolean success;
    String failReason;
    String conditionType;
}
```

### 3. MaterialChecker 材料检查器

**核心功能**：
- 检查玩家背包中的材料数量
- 支持 ItemCore 物品和原版物品
- 支持耐久度检查选项
- 提供材料消耗功能

**检查流程**：
```
checkMaterials(player, materials)
    ↓
遍历每个材料引用
    ↓
解析物品类型（ItemCore/Vanilla）
    ↓
统计玩家背包中的数量
    ↓
返回检查结果（包含每个材料的状态）
```

**材料状态**：
```java
MaterialStatus {
    ItemReference reference;
    int required;
    int available;
    boolean hasEnough;
}
```

### 4. CraftTask 制作任务

**数据结构**：
```java
CraftTask {
    UUID taskId;          // 任务唯一ID
    UUID playerId;        // 玩家UUID
    String forgeId;       // 锻造台ID
    String recipeId;      // 配方ID
    int amount;           // 制作数量
    long startTime;       // 开始时间
    long duration;        // 持续时间（毫秒）
    boolean cancelled;    // 是否已取消
    boolean completed;    // 是否已完成
}
```

**核心方法**：
- `getRemainingTime()` - 获取剩余时间
- `getRemainingSeconds()` - 获取剩余秒数
- `getProgress()` - 获取进度（0.0-1.0）
- `isComplete()` - 是否完成

### 5. RecipeQueue 制作队列

**数据结构**：
```java
RecipeQueue {
    int maxSize;              // 最大队列大小
    Queue<CraftTask> tasks;   // 等待队列
    CraftTask currentTask;    // 当前任务
}
```

**核心操作**：
- `addTask(CraftTask)` - 添加任务到队列
- `getCurrentTask()` - 获取当前执行的任务
- `completeCurrentTask()` - 完成当前任务，开始下一个
- `getQueueSize()` - 获取队列大小
- `cancelAll()` - 取消所有任务

### 6. CraftResult 制作结果

**结果类型**：
```java
enum ResultType {
    SUCCESS,               // 成功
    FAILURE,               // 失败
    CANCELLED,             // 已取消
    QUEUE_FULL,            // 队列已满
    MATERIALS_INSUFFICIENT,// 材料不足
    CONDITIONS_NOT_MET     // 条件不满足
}
```

**静态工厂方法**：
```java
CraftResult.success(task)
CraftResult.failure(reason)
CraftResult.cancelled(task)
CraftResult.queueFull()
CraftResult.materialsInsufficient(detail)
CraftResult.conditionsNotMet(detail)
```
