### 7. ItemCoreForge.java - 集成模块化架构

**文件**: `ItemCoreForge.java`

```java
public class ItemCoreForge extends JavaPlugin {
    private ForgeTypeManager typeManager;

    private void initManagers() {
        typeManager = new ForgeTypeManager(this);
    }

    private void loadConfigs() {
        // 加载配置...
        // 注册锻造台到类型处理器
        for (Forge forge : forgeLoader.getAllForges()) {
            typeManager.registerForgeToHandler(forge);
        }
    }

    @Override
    public void onDisable() {
        // 禁用所有类型处理器
        if (typeManager != null) {
            typeManager.disableAll();
        }
    }

    public void reload() {
        // 重载所有类型处理器
        if (typeManager != null) {
            typeManager.reloadAll();
        }
    }
}
```

## 📊 模块化架构的优势

| 优势 | 说明 |
|------|------|
| **独立部署** | 每个模块独立封装，bug 不会影响其他模块 |
| **易于扩展** | 新增类型只需新建一个 Handler 类 |
| **便于测试** | 可以单独测试每个模块 |
| **代码清晰** | 职责分离，主类不再臃肿 |
| **热插拔** | 可以动态注册/注销类型处理器 |

## 🚀 使用示例

### 1. 开启/关闭调试模式

```yaml
# config.yml
settings:
  debug: true   # 开启调试模式（测试阶段）
  # 或
  debug: false  # 关闭调试模式（生产环境）
```

### 2. 新增类型模块

要新增一个类型模块（如 `smithing_table`）：

```java
// 1. 创建类型处理器
public class SmithingTableHandler implements ForgeTypeHandler {
    @Override
    public String getType() {
        return "smithing_table";
    }
    // ... 实现接口方法
}

// 2. 在 ForgeTypeManager 中注册
public void init() {
    registerHandler(new CraftingTableHandler());
    registerHandler(new FurnaceHandler());
    registerHandler(new SmithingTableHandler());  // 新增
}
```
