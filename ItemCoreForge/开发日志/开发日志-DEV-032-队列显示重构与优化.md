## 🔧 问题分析

### 原有问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **队列显示混乱** | 任务完成后不会自动上移到第一个槽位 | 用户体验差 |
| **模型不一致** | QueueDisplayGUI 使用旧的 RecipeQueue/CraftTask | 代码维护困难 |
| **翻页冗余** | 队列只有6个槽位，翻页功能多余 | 增加不必要的复杂度 |
| **更新效率低** | 每秒全量刷新所有槽位 | 性能浪费 |

## 🔧 修改内容

### 1. QueueDisplayGUI - 模型统一

**文件**: `gui/QueueDisplayGUI.java`

**修改前**：使用旧的 `RecipeQueue` 和 `CraftTask`
```java
RecipeQueue queue = plugin.getPlayerQueueManager().getQueue(player, forge.getForgeId());
List<CraftTask> allTasks = queue.getAllTasks();
```

**修改后**：使用新的 `CraftingQueue` 和 `QueueItem`
```java
CraftingQueue queue = plugin.getCraftingQueueManager().getQueue(player, forge.getForgeId());
List<QueueItem> allItems = queue.getItems();
```

**改进点**:
- 移除对旧模型的依赖
- 使用绝对时间戳计算剩余时间
- 任务完成后自动从列表移除，后续任务自动上移

### 2. CustomForgeGUI - 队列显示优化

**文件**: `gui/CustomForgeGUI.java`

**优化的队列显示逻辑**:
```java
@Override
public void updateQueueDisplay() {
    if (layout == null || queueSlots.isEmpty()) return;
    
    CraftingQueue queue = plugin.getCraftingQueueManager().getQueue(player, forge.getForgeId());
    List<QueueItem> allItems = queue.getItems();
    
    // 按槽位顺序更新，任务自动上移
    for (int i = 0; i < queueSlots.size(); i++) {
        int slot = queueSlots.get(i);
        QueueItem item = null;
        if (i < allItems.size()) {
            item = allItems.get(i);  // 任务完成后自动上移
        }
        // 更新对应槽位的显示
        renderQueueDisplay(slot, config, item, i);
    }
}
```

**显示规则**:
| 队列索引 | 状态 | 显示内容 |
|---------|------|----------|
| 0（第一个） | 正在制作 | `&a正在制作` + 剩余时间 + 点击取消 |
| 0（第一个） | 已完成 | `&6已完成` + 点击领取 |
| 1+（后续） | 等待中 | `&e等待中` + 点击取消 |

### 3. CraftingQueue - 优化移除逻辑

**文件**: `core/CraftingQueue.java`

**改进的移除方法**:
```java
public void removeForClaim(UUID taskId) {
    QueueItem item = get(taskId);
    if (item != null) {
        item.setClaimed(true);
        items.remove(item);  // 移除后后续任务自动前移
    }
}
```

**时间调整逻辑**（取消任务时）:
```java
public void removeForCancel(UUID taskId, long craftingTimeMs) {
    int index = indexOf(taskId);
    if (index < 0) return;
    
    QueueItem removed = items.remove(index);
    removed.setCancelled(true);
    
    // 未完成任务取消时，后续任务提前完成
    if (!removed.isReady() && !removed.isClaimed()) {
        long savedTime = Math.min(removed.getRemaining(), craftingTimeMs);
        for (int i = index; i < items.size(); i++) {
            items.get(i).advanceCompletion(savedTime);
        }
    }
}
```

## 📊 新队列显示逻辑

### 队列流转示意图

```
初始队列: [A(制作中), B(等待), C(等待), D(等待), E(等待), F(等待)]
槽位显示:  [槽1:A,    槽2:B,   槽3:C,   槽4:D,   槽5:E,   槽6:F]

A完成并领取后:
队列: [B(制作中), C(等待), D(等待), E(等待), F(等待)]
槽位显示:  [槽1:B,    槽2:C,   槽3:D,   槽4:E,   槽5:F,   槽6:空]

B完成并领取后:
队列: [C(制作中), D(等待), E(等待), F(等待)]
槽位显示:  [槽1:C,    槽2:D,   槽3:E,   槽4:F,   槽5:空,   槽6:空]
```

### 核心优势

| 特性 | 原实现 | 新实现 |
|------|--------|--------|
| **任务上移** | ❌ 不会自动上移 | ✅ 完成后自动上移到第一个槽位 |
| **模型统一** | ❌ 新旧模型混用 | ✅ 统一使用 CraftingQueue/QueueItem |
| **时间计算** | 相对时间 | 绝对时间戳 |
| **持久化** | 重启后时间错误 | 重启后时间正确 |
