## 🔧 核心实现

### 1. 数据存储策略

**存储位置**: `plugins/ItemCoreForge/data/{player-uuid}.yml`

**触发时机**:
- 玩家离线时自动保存
- 服务器关闭时保存所有在线玩家数据
- 可手动调用保存

### 2. DataStorage 数据存储管理器

```java
public class DataStorage {
    
    // 保存玩家数据
    public void savePlayerData(Player player);
    
    // 加载玩家数据
    public void loadPlayerData(Player player);
    
    // 保存所有在线玩家数据
    public void saveAllData();
    
    // 清理过期数据（7天）
    public void cleanupOldData();
}
```

### 3. 数据格式

```yaml
player_id: "uuid-string"
saved_at: 1234567890
queues:
  - forge_id: "铁匠铺"
    tasks:
      - task_id: "task-uuid"
        forge_id: "铁匠铺"
        recipe_id: "iron_sword"
        amount: 1
        start_time: 1234567890
        duration: 5000
        completed: false
```

### 4. 生命周期集成

在 `ItemCoreForge.java` 中集成：

```java
// 玩家加入事件
@EventHandler
public void onPlayerJoin(PlayerJoinEvent event) {
    dataStorage.loadPlayerData(event.getPlayer());
}

// 玩家退出事件
@EventHandler
public void onPlayerQuit(PlayerQuitEvent event) {
    dataStorage.savePlayerData(event.getPlayer());
}

// 插件关闭
@Override
public void onDisable() {
    dataStorage.saveAllData();
}
```

---

### 4. 数据恢复优化 - 支持长时间制作任务

**问题描述**: 之前的实现保存的是 `start_time`，服务器重启后任务会从头开始计时。

**解决方案**: 
- 改为保存 `remaining_time`（剩余时间）
- 新增 `CraftTask` 构造函数支持恢复剩余时间
- 服务器重启后自动从剩余时间继续

**修改文件**: 
- `core/CraftTask.java` - 新增构造函数
- `storage/DataStorage.java` - 保存和恢复剩余时间

**数据格式**:
```yaml
queues:
  - forge_id: "铁匠铺"
    tasks:
      - task_id: "task-uuid"
        recipe_id: "iron_sword"
        amount: 1
        duration: 5000
        remaining_time: 3000  # 剩余3秒
```
