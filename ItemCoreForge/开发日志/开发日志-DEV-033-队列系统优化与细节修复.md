**修改文件**：`task/CraftingQueueManager.java`

### 五、取消任务时间计算 Bug 修复

**现象**：取消第一个任务后，第二个任务的制作时间翻倍（没有提前）

**原因**：`CraftingQueue.removeForCancel()` 中 `setCancelled(true)` 在 `getRemaining()` 计算之前被调用，而 `getRemaining()` 在 `cancelled=true` 时返回 0，导致 `savedTime=0`，后续任务时间未提前

**修复**：

```java
// 修复前：setCancelled 先执行，getRemaining() 返回 0
removed.setCancelled(true);   // ← 问题！
long savedTime = Math.min(removed.getRemaining(), craftingTimeMs); // = 0

// 修复后：先计算 savedTime，再设置 cancelled
long savedTime = Math.min(removed.getRemaining(), craftingTimeMs);
removed.setCancelled(true);   // 正确
```

**修改文件**：`core/CraftingQueue.java`

### 九、数据持久化三重重伤 — 已修复

**现象**：关闭服务器再重启后，队列中所有物品状态异常

**原因**：三个独立 bug 叠加导致持久化完全失效。

| # | Bug | 位置 | 影响 |
|---|-----|------|------|
| 1 | `queue.getItems().add(item)` | `DataStorage:L107` | `getItems()` 返回不可修改列表 → 加载必定抛 `UnsupportedOperationException` |
| 2 | `shutdown()` 清空队列后才 `saveAllData()` | `ItemCoreForge:L52-57` | 关闭时保存的是空数据 → 队列永久丢失 |
| 3 | 等待中的物品也用绝对时间戳恢复 | `DataStorage:L98-L104` | 服务器关闭期间所有物品时间都"过去了" → 全部显示已完成 |

**修复 1**：添加 `CraftingQueue.addRestored()` 方法供恢复时使用

**修复 2**：调换 `onDisable` 顺序，先保存再销毁

**修复 3**：加载后修复等待中物品的时间戳逻辑：
```
服务器关闭前：[A-制作中(剩余2s)] [B-等待中(5s)] [C-等待中(5s)]
服务器关闭10秒后重启：
  A 得到 2s 时间 → 已完成 ✓
  B 从 now 重新开始 5s 制作 → 不偷时间 ✓  
  C 仍在 B 后面等待 → 不偷时间 ✓
```

**修改文件**：
- `core/CraftingQueue.java` — 添加 `addRestored()`
- `storage/DataStorage.java` — 修复加载逻辑和时间调整
- `ItemCoreForge.java` — 调换 `onDisable` 顺序

### 六、领取物品双倍发放 Bug 修复

**现象**：领取已完成物品时，玩家获得双倍数量

**原因**：`canGiveOutput()` 中使用了 `inventory.addItem(output)` 来检查背包空间，但 `addItem()` 在检查空间时会**实际添加**物品到背包，导致物品被添加。后续 `giveOutput()` 再次添加，造成双倍

**修复**：

```java
// 修复前：addItem 会实际添加物品
HashMap<Integer, ItemStack> leftover = inventory.addItem(output); // ← 物品已被添加！
return leftover.isEmpty();

// 修复后：手动遍历检查空间，不修改背包
int maxStack = output.getMaxStackSize();
for (int i = 0; i < inventory.getSize(); i++) {
    ItemStack item = inventory.getItem(i);
    if (item == null || item.getType() == Material.AIR) {
        needed -= maxStack;  // 使用动态最大堆叠数
    } else if (item.isSimilar(output)) {
        needed -= (item.getMaxStackSize() - item.getAmount());
    }
    if (needed <= 0) return true;
}
return false;
```

**附加修复**：将硬编码的 `64`（空槽位容量）改为 `output.getMaxStackSize()`，兼容最大堆叠数不为 64 的物品（如工具/武器最大堆叠 1）

**修改文件**：`task/CraftingQueueManager.java`

### 七、队列显示逻辑二次修正

**第一次修正**：使用 `item.isCrafting()` 判断 → **失败**，因为所有等待中的任务 `isCrafting()` 也返回 true

**第二次修正**：使用 `slotIndex == 0` 判断 → **失败**，领取第一个任务后第二个任务仍在 slotIndex=1，不显示为"制作中"

**最终修正**：添加 `isFirstActiveTask(int slotIndex)` 方法，遍历前面所有任务，只要有一个未完成（`!isCompleted()`），当前任务就是"等待中"；否则是"正在制作"：

```java
private boolean isFirstActiveTask(int slotIndex) {
    CraftingQueue queue = ...;
    List<QueueItem> items = queue.getItems();
    for (int i = 0; i < slotIndex; i++) {
        if (i < items.size()) {
            QueueItem prevItem = items.get(i);
            if (!prevItem.isCompleted()) {  // isCompleted = isReady || claimed
                return false;
            }
        }
    }
    return true;
}
```

**修改文件**：`gui/CustomForgeGUI.java`、`gui/QueueDisplayGUI.java`

## ⚠️ 已知问题

### 背包满时仍可领取 — 已修复

**根本原因**：`inventory.getSize()` 返回 41（36 主背包 + 4 盔甲 + 1 副手），空盔甲/副手槽返回 `Material.AIR`，被误判为可用空间。

**对比**：
| | 旧系统 (MaterialChecker) | 新系统 (原 canGiveOutput) |
|---|---|---|
| 遍历方式 | `firstEmpty()` / `getContents()` | `inventory.getSize()` |
| 检查范围 | 仅主背包 36 格 | 41 格（含盔甲+副手） |
| 空盔甲槽 | 不参与检查 | **被当作空槽位** |

**修复**：将 `inventory.getSize()` 改为硬编码 `36`（仅主背包），与旧系统 `getContents()` 行为一致

### 八、背包满时领取失败后新物品立即完成 — 已修复

**现象**：背包满时领取已完成物品失败，之后添加的新物品不需要制作时间（立即完成）

**复现步骤**：
1. 队列：[A-已完成(30秒前), B-已完成(25秒前)]
2. 背包满，点击领取 A → 失败
3. 添加新物品 C (5秒制作)
4. C 立即完成（0秒）

**原因**：`CraftingQueue.add()` 计算新任务完成时间的公式是 `最后任务完成时间 + 制作时间`。当所有任务都已完成（完成时间在过去），新任务的完成时间 = 过去时间 + 5秒，可能仍在过去 → 立即完成

```java
// 修复前
completion = items.get(last).getCompletion() + craftingTimeMs;
// 如果最后一个任务30秒前就完成了 → completion = (now-30s) + 5s = now-25s < now → 立即完成！

// 修复后
completion = Math.max(items.get(last).getCompletion(), System.currentTimeMillis()) + craftingTimeMs;
```

**修改文件**：`core/CraftingQueue.java`
