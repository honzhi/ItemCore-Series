# ItemCoreForge 项目文档

## 项目概述

**项目名称**: ItemCoreForge  
**开发时间**: 2026-05-26 ~ 2026-05-28  
**环境**: Paper 1.21+, Java 17  
**核心功能**: Minecraft 服务器插件，提供自定义锻造台、配方管理、异步制作队列等功能

## 核心数据模型

### 1. Forge (锻造台)

```java
public class Forge {
    private String forgeId;           // 唯一标识
    private String displayName;       // 显示名称
    private String type;              // custom / crafting_table / furnace
    private int maxQueueSize;         // 最大队列大小
    private String layoutFile;        // GUI布局文件
    private boolean exactPlacement;   // 精确摆放(仅crafting_table)
    private Map<String, Recipe> recipes; // 配方映射
}
```

**三种类型对比**:

| 特性 | custom | crafting_table | furnace |
|------|--------|----------------|---------|
| GUI界面 | ✅ | ❌ | ❌ |
| 队列系统 | ✅ | ❌ | ❌ |
| 制作时间 | ✅ | ❌ | ✅(cook-time) |
| 条件检查 | ✅ | ✅ | ❌ |
| 命令打开 | ✅ `/icf gui` | ❌ | ❌ |
| 配置方式 | YAML布局 | 原版工作台 | 原版熔炉 |

### 2. Recipe (配方)

```java
public static class Recipe {
    private String recipeId;
    private ItemReference input;      // furnace类型使用
    private ItemReference output;
    private List<ItemReference> materials; // custom/crafting_table使用
    private List<Condition> conditions;
    private double craftTime;         // 秒
    private int cookTime;             // tick(熔炉专用)
    private Triggers triggers;        // 触发器配置
}
```

### 3. ItemReference (物品引用)

```java
// 格式: "source:category/id*amount" 或 "source:id*amount"
// 示例:
// - "itemcore:weapons/iron_sword*1"
// - "vanilla:IRON_INGOT*2"

public class ItemReference {
    private String source;     // itemcore / vanilla
    private String category;   // 仅itemcore
    private String id;
    private int amount;
    private boolean checkDurability;
    
    ItemStack toItemStack();   // 解析为物品
    boolean isItemCore();
}
```

### 4. CraftingQueue (新版队列)

```java
public class CraftingQueue {
    private String forgeId;
    private List<QueueItem> items = new ArrayList<>();
    private int maxSize;
    
    QueueItem add(String recipeId, int amount, long craftingTimeMs);
    void removeForCancel(UUID taskId);    // 取消任务,调整后续时间
    void removeForClaim(UUID taskId);     // 领取任务,调整后续时间
    List<QueueItem> getItems();
    QueueItem getItem(UUID taskId);
}
```

### 5. QueueItem (队列项)

```java
public class QueueItem {
    private UUID taskId;
    private String forgeId;
    private String recipeId;
    private int amount;
    private long start;           // 绝对时间戳
    private long completion;      // 绝对时间戳(预计完成)
    private boolean claimed;
    private boolean cancelled;
    
    boolean isCrafting();         // 正在制作
    boolean isReady();            // 已完成可领取
    boolean isClaimed();
    boolean isCancelled();
    double getRemainingSeconds();
    double getProgress();
    void advanceCompletion(long ms); // 提前完成时间
}
```

### 6. 任务状态流转

```
加入队列 → CRAFTING (制作中)
    ↓ 制作完成
→ READY (已完成,显示"点击领取")
    ↓ 点击领取
→ CLAIMED (已领取,从队列移除)

取消: CRAFTING/WAITING → CANCELLED → 从队列移除,返还材料
```

## 队列系统核心设计

### 时间计算逻辑

```java
// 新增任务时
long start = System.currentTimeMillis();
long completion = items.isEmpty()
    ? start + craftingTimeMs                             // 第一个任务
    : Math.max(items.get(last).getCompletion(), start)   // 后续任务
        + craftingTimeMs;
```

### 取消任务时间调整

```
队列: [A(10s), B(5s), C(8s)]
完成时间: A=10s, B=15s, C=23s

等待3秒后取消A:
- A剩余时间 = 7s
- 节省时间 = 7s
- B完成时间 = 15s - 7s = 8s
- C完成时间 = 23s - 7s = 16s
```

### 数据持久化

**存储位置**: `plugins/ItemCoreForge/data/{uuid}.yml`

```yaml
player_id: "xxxx-xxxx-xxxx"
saved_at: 1716888000000
queues:
  - forge_id: "example_template"
    items:
      - task_id: "xxxx-xxxx-xxxx"
        recipe_id: "iron_sword"
        amount: 1
        start: 1716888000000
        completion: 1716888005000
```

**恢复逻辑**:
- 加载时检查 `completion <= now`: 标记为 READY
- 加载时检查 `start > now`: 重新计算 completion (处理等待中任务)

## ItemCore软依赖

### plugin.yml配置

```yaml
softdepend: [ItemCore]
```

### 依赖检查

```java
// ItemCoreForge.onEnable()
if (!checkItemCore()) {
    getLogger().warning("未检测到 ItemCore 插件，将仅支持 vanilla 物品！");
}
```

### API调用方式

所有ItemCore API调用通过反射实现，不直接import。

```java
// MaterialChecker.resolveItem()
if (ref.isItemCore()) {
    Class<?> apiClass = Class.forName("com.minemart.itemcore.api.ItemCoreAPI");
    Object api = apiClass.getMethod("getInstance").invoke(null);
    return (ItemStack) apiClass.getMethod("getItem", String.class)
        .invoke(api, ref.getCategory() + "/" + ref.getId());
}
```

### 依赖状态影响

| 场景 | 行为 |
|------|------|
| 有ItemCore | 全部功能正常，支持itemcore和vanilla |
| 无ItemCore | 正常启动，仅支持vanilla，itemcore配方自动跳过 |

```
ItemCoreForge (主类)
    └── ForgeTypeManager
            ├── CraftingTableHandler
            │       └── CraftingTableRecipeManager
            └── FurnaceHandler
                    └── FurnaceRecipeManager
```

### crafting_table实现要点

1. 注册"基础配方"(每槽1个)到Bukkit
2. 拦截CraftItemEvent,取消原版行为
3. 自己计算最大可制作次数(支持Shift点击)
4. 手动扣除材料(支持堆叠)
5. 手动给予输出物品

### furnace实现要点

1. 注册FurnaceRecipe到Bukkit
2. 支持cook-time配置(默认200tick)
3. 不支持条件检查(无法确定操作玩家)

## 制作流程 (custom类型)

```
玩家点击配方选择 → 打开CustomForgeGUI
    ↓
点击确认按钮
    ↓
┌─────────────────────────────────────┐
│  前置检查                            │
│  ├── 材料检查 (MaterialChecker)     │
│  └── 条件检查 (ConditionChecker)    │
└─────────────────────────────────────┘
    ↓ 检查通过
消耗材料
    ↓
创建QueueItem,加入CraftingQueue
    ↓
触发 RecipeQueueEvent,更新GUI
    ↓ 异步等待制作完成
CraftingQueueManager每秒检查:
  if (item.getCompletion() <= now && !item.isClaimed() && !item.isCancelled())
      标记为READY
    ↓
玩家点击领取
    ↓
检查背包空间
    ↓
给予物品
    ↓
从队列移除,调整后续任务时间
    ↓
更新GUI
```

## 关键Bug修复记录

### 1. 双倍领取Bug

**问题**: `canGiveOutput()` 使用 `inventory.addItem()` 检查空间,实际添加了物品。

**修复**: 手动遍历背包槽位计算可用空间,不修改背包。

### 2. 取消任务时间翻倍

**问题**: `setCancelled(true)` 在 `getRemaining()` 之前调用,导致 savedTime=0。

**修复**: 先计算 savedTime,再设置 cancelled。

### 3. 数据持久化三重Bug

**问题1**: `queue.getItems().add()` 返回不可修改列表。

**问题2**: `onDisable()` 先清空队列再保存,导致保存空数据。

**问题3**: 等待中任务使用绝对时间戳恢复,服务器重启后全部显示已完成。

**修复**:
1. 添加 `CraftingQueue.addRestored()` 方法
2. 调换顺序: 先saveAllData(),再shutdown()
3. 加载后修复等待中任务时间戳

### 4. 背包满后新任务立即完成

**问题**: `add()` 计算 completion 时未考虑 "已完成但未领取" 的任务。

**修复**: `completion = Math.max(lastCompletion, now) + craftingTimeMs`
