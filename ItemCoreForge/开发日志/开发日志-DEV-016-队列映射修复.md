## 📋 开发目标

修复 CustomForgeGUI 中队列显示槽位的映射问题，确保队列任务按照正确的顺序映射到布局中定义的 `queue_display` 槽位上。

---

## 🔧 问题描述

**问题**: 队列任务没有按照预期的顺序映射到布局中的 `queue_display` 槽位上。

**原因分析**:
- 原来的实现中，`queue_display` 槽位是按照布局遍历顺序收集的
- 如果布局文件中的槽位不是连续的（例如分散在不同位置），映射顺序就会混乱
- 缺少对槽位的排序处理

---

## 🛠 修复内容

### 修改文件

**文件**: `gui/CustomForgeGUI.java`

#### 修改 `renderLayout()` 方法

```java
// 修复前
private void renderLayout() {
    for (int row = 0; row < layout.getRows(); row++) {
        for (int col = 0; col < 9; col++) {
            int slot = row * 9 + col;
            // ... 直接收集槽位
            if ("queue_display".equals(function)) {
                queueSlots.add(slot);
            }
        }
    }
}

// 修复后
private void renderLayout() {
    // 先收集所有 queue_display 槽位
    List<Integer> tempQueueSlots = new ArrayList<>();
    
    for (int row = 0; row < layout.getRows(); row++) {
        for (int col = 0; col < 9; col++) {
            int slot = row * 9 + col;
            char c = layout.getSlot(row, col);
            LayoutLoader.SlotConfig config = layout.getSlotConfig(c);
            
            if (config != null && "queue_display".equals(config.getFunction())) {
                tempQueueSlots.add(slot);
            }
        }
    }
    
    // 按槽位号排序（确保从左到右、从上到下的视觉顺序）
    tempQueueSlots.sort(Integer::compareTo);
    queueSlots.addAll(tempQueueSlots);
    
    // 然后渲染布局
    // ...
}
```

#### 修改 `renderSlot()` 方法

```java
// 修复前
private void renderSlot(int slot, LayoutLoader.SlotConfig config) {
    // ...
    if ("queue_display".equals(function)) {
        queueSlots.add(slot);  // 移除这个收集逻辑
    }
    
    switch (function) {
        // ...
        case "queue_display" -> renderQueueDisplay(slot, config, queueSlots.size() - 1);
    }
}

// 修复后
private void renderSlot(int slot, LayoutLoader.SlotConfig config) {
    // ...
    switch (function) {
        // ...
        case "queue_display" -> renderQueueDisplay(slot, config, queueSlots.indexOf(slot));
    }
}
```

## ✅ 修复效果

### 映射规则

| 队列索引 | 队列状态 | 映射位置 |
|----------|----------|----------|
| 0 | 正在制作 | 最小槽位号的 queue_display |
| 1 | 等待中 | 次小槽位号的 queue_display |
| 2 | 等待中 | 第三小槽位号的 queue_display |
| ... | ... | ... |

### 示例

假设布局中定义了以下 `queue_display` 槽位：`{10, 3, 19, 28}`

修复前（按遍历顺序）:
```
槽位 3 → 队列索引 0 (正在制作)
槽位 10 → 队列索引 1 (等待中)  
槽位 19 → 队列索引 2 (等待中)
槽位 28 → 队列索引 3 (等待中)
```

修复后（按槽位号排序）:
```
槽位 3 → 队列索引 0 (正在制作)
槽位 10 → 队列索引 1 (等待中)
槽位 19 → 队列索引 2 (等待中)
槽位 28 → 队列索引 3 (等待中)
```

### 改进点

1. **收集阶段**: 先遍历布局收集所有 `queue_display` 槽位
2. **排序阶段**: 使用 `Integer::compareTo` 按槽位号排序
3. **渲染阶段**: 使用 `queueSlots.indexOf(slot)` 获取正确的队列索引
