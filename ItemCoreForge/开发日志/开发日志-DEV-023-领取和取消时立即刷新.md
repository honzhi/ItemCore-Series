# 开发日志 - DEV-023 - 领取和取消时立即刷新

**开发日期**: 2026-05-27  
**开发阶段**: Phase 23  
**开发状态**: ✅ 已完成

---

## 📋 开发目标

在领取物品和取消制作后立即刷新队列显示，避免出现 1 秒的延迟。

---

## 🔧 问题分析

### 问题描述

玩家点击领取或取消后，队列中的物品依然会存在 1 秒才消失。

### 原由分析

#### 原有更新机制

在 `CraftScheduler` 中已经有更新调用：

```java
// CraftScheduler.cancelCraft()
queue.cancelTask(taskId);
materialChecker.giveMaterials(player, recipe.getMaterials());
// ...
plugin.getGuiListener().updatePlayerQueueDisplay(player);  // 有调用

// CraftScheduler.claimItem()
giveOutput(player, recipe);
queue.removeClaimedTask(taskId);
// ...
plugin.getGuiListener().updatePlayerQueueDisplay(player);  // 有调用
```

但问题在于：

1. `GuiListener.updatePlayerQueueDisplay()` 是通过 `openMenus` Map 获取菜单实例
2. 可能存在时序问题，或者 GUI 中的方法没有直接触发刷新
3. 依赖 `CraftScheduler` 中的调用不如在 GUI 中直接调用可靠

#### 对比开始制作时的实现

在 `CustomForgeGUI.startCraft()` 中：

```java
if (result.isSuccess()) {
    updateQueueDisplay();  // 直接调用刷新
}
```

这种方式更直接、更可靠。

---

## 🛠 解决方案

在 GUI 层的 `claimItem()` 和 `cancelTask()` 方法中，成功后直接调用刷新方法。

### 修改 CustomForgeGUI

**文件**: `gui/CustomForgeGUI.java`

```java
// 修复前
private void claimItem(CraftTask task) {
    CraftResult result = plugin.getCraftScheduler().claimItem(...);
    if (!result.isSuccess()) {
        MessageUtil.sendWithPrefix(player, "&c" + result.getMessage());
    }
}

private void cancelTask(CraftTask task) {
    CraftResult result = plugin.getCraftScheduler().cancelCraft(...);
    if (!result.isSuccess()) {
        MessageUtil.sendWithPrefix(player, "&c" + result.getMessage());
    }
}

// 修复后
private void claimItem(CraftTask task) {
    CraftResult result = plugin.getCraftScheduler().claimItem(...);
    if (result.isSuccess()) {
        updateQueueDisplay();  // 立即刷新
    } else {
        MessageUtil.sendWithPrefix(player, "&c" + result.getMessage());
    }
}

private void cancelTask(CraftTask task) {
    CraftResult result = plugin.getCraftScheduler().cancelCraft(...);
    if (result.isSuccess()) {
        updateQueueDisplay();  // 立即刷新
    } else {
        MessageUtil.sendWithPrefix(player, "&c" + result.getMessage());
    }
}
```

### 修改 QueueDisplayGUI

**文件**: `gui/QueueDisplayGUI.java`

## 📊 完整更新机制总结

现在队列会在以下时机立即更新：

| 操作 | 位置 | 更新方式 |
|------|------|----------|
| **开始制作** | `CustomForgeGUI.startCraft()` | 调用 `updateQueueDisplay()` |
| **制作完成** | `CraftScheduler.executeCraft()` | 调用 `updatePlayerQueueDisplay()` |
| **取消制作** | `CustomForgeGUI.cancelTask()` | 调用 `updateQueueDisplay()` |
| **取消制作** | `QueueDisplayGUI.cancelTask()` | 调用 `render()` |
| **领取物品** | `CustomForgeGUI.claimItem()` | 调用 `updateQueueDisplay()` |
| **领取物品** | `QueueDisplayGUI.claimItem()` | 调用 `render()` |
| **定时刷新** | GUI 的定时任务 | 每 1 秒刷新一次（用于剩余时间同步） |
