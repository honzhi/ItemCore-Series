## 🔧 问题修复

### 1. RecipePreCraftEvent 编译错误

**问题描述**: `RecipePreCraftEvent` 类中的 `isCancelled()` 和 `setCancelled()` 方法无法正确覆盖父类接口。

**错误信息**:
```
方法不会覆盖或实现超类型的方法
```

**解决方案**: 
- 让 `RecipePreCraftEvent` 实现 `Cancellable` 接口
- 修复方法签名以匹配接口要求

**修改文件**: `api/event/RecipePreCraftEvent.java`

```java
// 修改前
public class RecipePreCraftEvent extends Event {

// 修改后
public class RecipePreCraftEvent extends Event implements Cancellable {
```

### 2. pom.xml 构建配置优化

**问题描述**: Maven shade 插件配置不正确，导致 JAR 包生成失败。

**解决方案**: 移除 shade 插件，使用标准 jar 插件。

**修改文件**: `pom.xml`

### 3. DataStorage 方法调用错误

**问题描述**: `DataStorage.java` 调用了不存在的方法。

**错误信息**:
```
找不到符号: 方法 getTasks()
找不到符号: 方法 getRecipeById(java.lang.String)
```

**解决方案**: 
- 将 `queue.getTasks()` 改为 `queue.getAllTasks()`
- 将 `forge.getRecipeById()` 改为 `forge.getRecipe()`

**修改文件**: `storage/DataStorage.java`

### 4. GUI布局不显示问题

**问题描述**: 点击配方后，锻造台GUI不显示任何布局内容。

**根本原因**: 
- `LayoutLoader` 存储布局时使用不带扩展名的名称（如 "Default"）
- `CustomForgeGUI` 获取布局时传入的是完整文件名（如 "Default.yml"）
- 导致布局查找失败，`layout` 为 null

**解决方案**: 在 `CustomForgeGUI.java` 中移除文件名扩展名

**修改文件**: `gui/CustomForgeGUI.java`

```java
// 修改前
this.layout = plugin.getLayoutLoader().getLayout(forge.getLayoutFile());

// 修改后
String layoutName = forge.getLayoutFile().replace(".yml", "");
this.layout = plugin.getLayoutLoader().getLayout(layoutName);
```
