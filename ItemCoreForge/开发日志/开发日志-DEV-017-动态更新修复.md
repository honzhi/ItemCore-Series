## 📋 开发目标

修复 CustomForgeGUI 中队列动态更新的问题，确保队列状态实时正确显示。

---

## 🔧 问题描述

**用户反馈的问题**:
1. 进入队列后需要退出配方 GUI，再进入才会显示
2. 过 1 秒后队列又全部消失了
3. 但在总队列（QueueDisplayGUI）中显示正常

**原因分析**:

### 问题 1: updateQueueDisplay() 使用错误的配置

```java
// 修复前
private void updateQueueDisplay() {
    // 只获取第一个 queue_display 的配置
    LayoutLoader.SlotConfig queueConfig = null;
    for (LayoutLoader.SlotConfig config : layout.getSlotConfigs().values()) {
        if ("queue_display".equals(config.getFunction())) {
            queueConfig = config;
            break;  // 只取第一个！
        }
    }
    
    // 然后用同一个配置渲染所有槽位
    for (int i = 0; i < queueSlots.size(); i++) {
        int slot = queueSlots.get(i);
        renderQueueDisplay(slot, queueConfig, i);  // 所有槽位用同一个 config！
    }
}
```

**问题**: 如果第一个 `queue_display` 槽位的 Material 是 AIR 或者有特殊配置，更新时所有槽位都会被错误地渲染。

### 问题 2: renderLayout() 重复添加槽位

```java
// 修复前
private void renderLayout() {
    List<Integer> tempQueueSlots = new ArrayList<>();
    // ... 收集槽位
    
    tempQueueSlots.sort(Integer::compareTo);
    queueSlots.addAll(tempQueueSlots);  // 没有先清空！
    
    // ... 渲染
}
```

**问题**: 
- 构造函数中调用一次 `renderLayout()`
- `selectRecipe()` 或 `update()` 也会调用 `renderLayout()`
- 每次调用都用 `addAll()` 追加，导致 `queueSlots` 越来越大
- 最终 `queueSlots.indexOf(slot)` 可能返回错误的索引

---

## 🛠 修复内容

### 修改文件

**文件**: `gui/CustomForgeGUI.java`

#### 修复 1: updateQueueDisplay() 为每个槽位获取正确配置

```java
// 修复后
private void updateQueueDisplay() {
    if (layout == null) return;
    
    for (int i = 0; i < queueSlots.size(); i++) {
        int slot = queueSlots.get(i);
        // 为每个槽位获取对应的配置
        LayoutLoader.SlotConfig config = getSlotConfigForSlot(slot);
        if (config != null) {
            renderQueueDisplay(slot, config, i);
        }
    }
}
```

#### 修复 2: renderLayout() 添加前先清空

```java
// 修复后
private void renderLayout() {
    List<Integer> tempQueueSlots = new ArrayList<>();
    // ... 收集槽位
    
    tempQueueSlots.sort(Integer::compareTo);
    queueSlots.clear();  // 先清空
    queueSlots.addAll(tempQueueSlots);  // 然后添加
    
    // ... 渲染
}
```

## ✅ 修复效果

### 修复前

1. 初始渲染正常（每个槽位使用自己的配置）
2. 1 秒后 `updateQueueDisplay()` 执行，所有槽位使用第一个槽位的配置
3. 如果第一个槽位的 Material 是 AIR，所有队列显示都消失

### 修复后

1. 初始渲染正常
2. `updateQueueDisplay()` 为每个槽位获取正确的配置
3. 队列状态实时正确显示
