# 开发日志 - DEV-020 - 制作时立即刷新修复

**开发日期**: 2026-05-27  
**开发阶段**: Phase 20  
**开发状态**: ✅ 已完成

---

## 📋 开发目标

确保开始制作时能够立即刷新队列显示，而不是等待 1 秒的定时刷新。

---

## 🔧 问题描述

**用户反馈**: 没有做到开始制作时立即刷新队列。

**原因分析**:

虽然实现了事件驱动的更新机制，但在某些情况下事件可能没有正确触发或处理。

### 原有实现

```java
// CustomForgeGUI.startCraft()
private void startCraft() {
    if (selectedRecipe == null) {
        return;
    }
    
    CraftResult result = plugin.getCraftScheduler().enqueueCraft(
        player, 
        forge.getForgeId(), 
        selectedRecipe.getRecipeId()
    );
    
    if (!result.isSuccess()) {
        MessageUtil.sendWithPrefix(player, "&c" + result.getMessage());
    }
}
```

**问题**:
- 调用 `enqueueCraft()` 后，依赖 `RecipeQueueEvent` 事件触发更新
- 事件系统可能存在时序问题或配置问题
- 没有直接的立即更新机制

---

## 🛠 修复方案

采用 **直接调用 + 事件驱动** 的双重保障机制：

1. **直接调用**: 在 `startCraft()` 成功后立即调用 `updateQueueDisplay()`
2. **事件驱动**: 保留事件系统，用于其他插件监听和跨场景更新

---

## 📝 具体实现

### 修改文件

**文件**: `gui/CustomForgeGUI.java`

#### 修改 `startCraft()` 方法

```java
// 修复前
private void startCraft() {
    if (selectedRecipe == null) {
        return;
    }
    
    CraftResult result = plugin.getCraftScheduler().enqueueCraft(
        player, 
        forge.getForgeId(), 
        selectedRecipe.getRecipeId()
    );
    
    if (!result.isSuccess()) {
        MessageUtil.sendWithPrefix(player, "&c" + result.getMessage());
    }
}

// 修复后
private void startCraft() {
    if (selectedRecipe == null) {
        return;
    }
    
    CraftResult result = plugin.getCraftScheduler().enqueueCraft(
        player, 
        forge.getForgeId(), 
        selectedRecipe.getRecipeId()
    );
    
    if (result.isSuccess()) {
        updateQueueDisplay();  // 立即刷新队列显示
    } else {
        MessageUtil.sendWithPrefix(player, "&c" + result.getMessage());
    }
}
```

## 📊 更新机制对比

### 方案 1: 仅事件驱动

```
点击制作
    ↓
enqueueCraft()
    ↓
触发 RecipeQueueEvent
    ↓
GuiListener 监听事件
    ↓
调用 updateQueueDisplay()
    ↓
刷新显示
```

**问题**: 事件系统可能存在问题

### 方案 2: 仅直接调用（当前采用）

```
点击制作
    ↓
enqueueCraft()
    ↓
成功? ──是──→ updateQueueDisplay() 立即刷新
    ↓ 否
显示错误消息
```

**优点**: 可靠，没有时序问题

### 方案 3: 双重保障（推荐）

```
点击制作
    ↓
enqueueCraft()
    ↓
成功? ──是──→ 1. 触发 RecipeQueueEvent（其他插件用）
    │             2. 调用 updateQueueDisplay()（立即刷新）
    ↓ 否
显示错误消息
```

**优点**: 兼顾可靠性和扩展性
