## 📋 开发目标

修复配方队列中等待槽位不显示的问题，确保所有 `queue_display` 槽位在没有物品时都能显示配置的 `display-name` 和 `lore`。

---

## 🔧 问题描述

**用户反馈**: 在配方的队列里，只有第一个槽位会显示 lore，剩余的等待槽位不显示（队列里没有物品的时候）。

**原因分析**:

查看 `renderQueueDisplay()` 方法的逻辑：

```java
// 修复前
if (task == null) {
    ItemStack item = new ItemStack(config.getMaterial());
    ItemMeta meta = item.getItemMeta();
    if (queueIndex == 0) {  // ❌ 只有第一个槽位
        meta.displayName(MessageUtil.toComponent(config.getDisplayName()));
        List<Component> lore = new ArrayList<>();
        lore.add(MessageUtil.toComponent("&7当前没有正在制作的物品"));
        meta.lore(lore);
    } else {
        meta.displayName(MessageUtil.toComponent(""));  // ❌ 其他槽位什么都不显示
    }
    item.setItemMeta(meta);
    setItem(slot, item);
}
```

**问题**:
1. 只有 `queueIndex == 0`（第一个槽位）才会设置 displayName 和 lore
2. 其他槽位（等待槽位）只显示空名称，没有使用配置中的内容
3. 硬编码了 lore 内容（"&7当前没有正在制作的物品"），而不是使用配置文件中的 lore

---

## 🛠 修复内容

### 修改文件

**文件**: `gui/CustomForgeGUI.java`

#### 修改 `renderQueueDisplay()` 方法

```java
// 修复前
if (task == null) {
    ItemStack item = new ItemStack(config.getMaterial());
    ItemMeta meta = item.getItemMeta();
    if (queueIndex == 0) {
        meta.displayName(MessageUtil.toComponent(config.getDisplayName()));
        List<Component> lore = new ArrayList<>();
        lore.add(MessageUtil.toComponent("&7当前没有正在制作的物品"));
        meta.lore(lore);
    } else {
        meta.displayName(MessageUtil.toComponent(""));
    }
    item.setItemMeta(meta);
    setItem(slot, item);
}

// 修复后
if (task == null) {
    ItemStack item = new ItemStack(config.getMaterial());
    ItemMeta meta = item.getItemMeta();
    
    if (!config.getDisplayName().isEmpty()) {
        meta.displayName(MessageUtil.toComponent(config.getDisplayName()));
    }
    
    if (config.getLore() != null && !config.getLore().isEmpty()) {
        List<Component> lore = new ArrayList<>();
        for (String line : config.getLore()) {
            lore.add(MessageUtil.toComponent(line));
        }
        meta.lore(lore);
    }
    
    item.setItemMeta(meta);
    setItem(slot, item);
}
```

## 📊 改进点

1. **统一显示**: 所有 `queue_display` 槽位都使用配置中的 `display-name` 和 `lore`
2. **配置驱动**: 不再硬编码 lore 内容，完全使用配置文件中的设置
3. **灵活性**: 可以为不同的 queue_display 槽位配置不同的显示内容
