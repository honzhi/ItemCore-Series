## 🔧 问题分析

### 问题描述
1. 队列显示不显示物品
2. 点击制作后出现两次"开始制作"提示
3. 当前设计复杂，难以调试

### 根本原因
1. 队列显示逻辑与配方选择界面耦合太紧
2. 消息发送了两次（CraftScheduler 和 CustomForgeGUI 都发送了）
3. 布局系统与队列显示冲突

## 🛠 修复内容

### 1. 创建独立的队列显示 GUI

**新增文件**: `gui/QueueDisplayGUI.java`

```java
public class QueueDisplayGUI extends BaseMenu {
    
    public QueueDisplayGUI(ItemCoreForge plugin, Player player, Forge forge) {
        super(plugin, player, "制作队列 - " + forge.getDisplayName(), 6);
        this.forge = forge;
        render();
        startUpdateTask();
    }
    
    // 从左到右依次显示队列中的物品
    // 最多显示 54 个槽位（6行 GUI）
    private void render() {
        RecipeQueue queue = plugin.getPlayerQueueManager().getQueue(player, forge.getForgeId());
        List<CraftTask> allTasks = queue.getAllTasks();
        
        for (int i = 0; i < 54; i++) {
            if (i < allTasks.size()) {
                // 显示队列中的物品
                ItemStack item = createQueueItem(recipe, task, i);
                inventory.setItem(i, item);
            } else {
                // 显示空槽位
                inventory.setItem(i, createEmptySlot(i));
            }
        }
    }
}
```

**特点**:
- 独立 GUI 显示整个队列
- 从左到右依次映射队列（最多 54 槽位）
- 第一个槽位显示"正在制作"
- 其他槽位显示"等待 #N"
- 每秒自动更新剩余时间

### 3. 修复重复提示问题

**问题原因**: 
- `CraftScheduler.enqueueCraft()` 发送了 "crafting-started" 消息
- `CustomForgeGUI.startCraft()` 也发送了相同的消息

**解决方案**: 移除 CustomForgeGUI 中的消息发送

**修改文件**: `gui/CustomForgeGUI.java`

```java
private void startCraft() {
    CraftResult result = plugin.getCraftScheduler().enqueueCraft(
        player, 
        forge.getForgeId(), 
        selectedRecipe.getRecipeId()
    );
    
    // 只在失败时发送错误消息
    if (!result.isSuccess()) {
        MessageUtil.sendWithPrefix(player, "&c" + result.getMessage());
    }
    // 成功消息由 CraftScheduler 发送
}
```

## 📊 新架构说明

### 界面流程

```
配方选择界面 (RecipeSelectGUI)
    │
    ├── 点击配方 → 配方详情界面 (CustomForgeGUI)
    │                      │
    │                      └── 点击开始制作 → 开始制作（显示消息）
    │
    └── 点击队列按钮 → 队列显示界面 (QueueDisplayGUI)
                            │
                            └── 显示所有队列物品（从左到右映射）
```

### 队列显示逻辑

```
队列: [铁剑, 钻石剑, 金剑, ...]
           ↓
槽位 0: 正在制作 - 铁剑
槽位 1: 等待 #2 - 钻石剑
槽位 2: 等待 #3 - 金剑
...
槽位 53: (空)
```
