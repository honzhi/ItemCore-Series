## 🔧 核心实现

### 1. 主类结构

```java
public class ItemCoreForge extends JavaPlugin {
    // 单例模式
    private static ItemCoreForge instance;
    
    // 核心管理器
    private ConfigManager configManager;
    private ForgeLoader forgeLoader;
    private LayoutLoader layoutLoader;
    private ForgeManager forgeManager;
    private CraftScheduler craftScheduler;
    private PlayerQueueManager playerQueueManager;
    
    // 生命周期方法
    onEnable() -> 初始化管理器、加载配置、注册命令
    onDisable() -> 关闭调度器、保存数据
}
```

### 2. 配置加载流程

```
启动插件
    ↓
检查 ItemCore 依赖
    ↓
初始化各管理器
    ↓
加载 config.yml
    ↓
加载 forges/*.yml
    ↓
加载 layouts/*.yml
    ↓
注册命令和监听器
```

### 3. 数据模型

**Forge（锻造台）**:
- `forgeId`: 唯一标识
- `displayName`: 显示名称
- `maxQueueSize`: 最大队列大小
- `type`: 类型（custom/crafting_table/furnace/brewing_stand）
- `layoutFile`: 布局文件
- `recipes`: 配方列表

**Recipe（配方）**:
- `recipeId`: 配方ID
- `output`: 输出物品引用
- `materials`: 材料列表
- `conditions`: 条件列表
- `craftTime`: 制作时间

**ItemReference（物品引用）**:
- `source`: 来源（itemcore/vanilla）
- `category`: 分类（仅 itemcore）
- `id`: 物品ID
- `amount`: 数量
- `checkDurability`: 是否检查耐久

**Condition（条件）**:
- `type`: 类型（level/permission/money）
- `value`: 值
- `node`: 权限节点
- `amount`: 金额

### 4. GUI 布局系统

使用字符映射定义 GUI 布局：
```yaml
layout:
  - 'AAAAAAAAA'  # 边框
  - 'AAA0E0AAA'  # 材料 + 输出
  - 'AZ000000A'  # 确认 + 材料
  - 'AAA0X0AAA'  # 材料 + 队列
  - 'AAAAAAAAA'  # 边框
  - 'AXXXXXXXA'  # 队列信息
```

功能槽位：
- `border`: 装饰边框
- `material_slot`: 材料槽位
- `output_slot`: 输出物品显示
- `queue_display`: 制作队列显示（含剩余时间）
- `confirm_button`: 确认制作按钮
- `cancel_button`: 取消按钮
