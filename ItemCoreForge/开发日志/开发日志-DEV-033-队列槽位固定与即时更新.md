## 📋 需求分析

根据用户反馈，正确的队列行为应该是：

```
初始:      [任务A-制作中] [任务B-等待] [任务C-等待] [空] [空]
A完成:     [任务A-已完成] [任务B-制作中] [任务C-等待] [空] [空]
领取A:     [任务B-制作中] [任务C-等待] [空] [空] [空]
添加D:     [任务B-制作中] [任务C-等待] [任务D-等待] [空] [空]
```

### 核心需求

1. **任务顺序执行**：队列按顺序制作，第一个任务完成后第二个开始
2. **自动上移**：领取任务后，后续任务自动前移填补空位
3. **即时更新**：状态变化时立即更新 GUI，无延迟

## 🔧 实现方案

### 1. CraftingQueue - 队列管理

```java
public QueueItem add(String recipeId, int amount, long craftingTimeMs) {
    if (isFull()) return null;

    long start = System.currentTimeMillis();
    long completion = items.isEmpty()
        ? start + craftingTimeMs
        : items.get(items.size() - 1).getCompletion() + craftingTimeMs;

    QueueItem item = new QueueItem(forgeId, recipeId, amount, start, completion);
    items.add(item);
    return item;
}

public void removeForClaim(UUID taskId) {
    int index = indexOf(taskId);
    if (index >= 0) {
        QueueItem item = items.get(index);
        item.setClaimed(true);
        items.remove(index);
        
        // 更新后续任务的完成时间（提前）
        if (index < items.size()) {
            long savedTime = item.getCompletion() - System.currentTimeMillis();
            if (savedTime > 0) {
                for (int i = index; i < items.size(); i++) {
                    items.get(i).advanceCompletion(savedTime);
                }
            }
        }
    }
}
```

**关键逻辑**：
- 使用简单的 `List<QueueItem>` 存储，按顺序排列
- 新任务添加到列表末尾
- 领取任务后从列表中移除，后续任务自动前移
- 领取时计算节省的时间，后续任务提前完成

### 2. 队列流转示例

@Override

    for (int i = 0; i < queueSlots.size(); i++) {
        int slot = queueSlots.get(i);

        renderQueueDisplay(slot, config, item, i);
    }
