## 更新日期

## 更新内容

### 1. 配置重构

将全局配置中的 `default-success-rate` 和 `consume-on-fail` 移动到每个配方中，允许不同配方有不同的成功率设置。

**移除的全局配置** (`config.yml`):
```yaml
settings:
  default-success-rate: 100  # 已移除
  consume-on-fail: true      # 已移除
```

**新增的配方配置** (`forges/*.yml`):
```yaml
recipes:
  iron_sword:
    craft-time: 5.0
    success-rate: 100        # 新增：成功率（百分比）
    consume-on-fail: true    # 新增：失败时是否消耗材料
```

### 2. 代码修改

| 文件 | 修改内容 |
|------|----------|
| `Forge.java` | 在 `Recipe` 类中添加 `successRate` 和 `consumeOnFail` 字段 |
| `ForgeLoader.java` | 加载配方级别的 `success-rate` 和 `consume-on-fail` 配置 |
| `CraftScheduler.java` | 实现成功率判定逻辑 |
| `config.yml` | 移除全局的 `default-success-rate` 和 `consume-on-fail` |
| `example_template.yml` | 添加配方配置示例 |

### 3. 功能说明

**成功率判定逻辑**:
- 当 `success-rate` 为 100 时，必定成功
- 当 `success-rate` < 100 时，随机生成 1-100 的数字进行判定
- 如果随机数 > 成功率，则制作失败

**材料消耗逻辑**:
- 制作成功：材料正常消耗
- 制作失败 + `consume-on-fail: true`：材料被消耗
- 制作失败 + `consume-on-fail: false`：材料返还给玩家

### 4. 使用示例

```yaml
recipes:
  # 高难度配方：50%成功率，失败返还材料
  legendary_sword:
    output:
      source: "itemcore"
      category: "weapons"
      id: "legendary_sword"
      amount: 1
    
    materials:
      - source: "itemcore"
        category: "materials"
        id: "legendary_ingot"
        amount: 5
    
    craft-time: 30.0
    success-rate: 50        # 50%成功率
    consume-on-fail: false  # 失败时返还材料
  
  # 普通配方：100%成功率
  iron_sword:
    output:
      source: "vanilla"
      id: "IRON_SWORD"
      amount: 1
    
    materials:
      - source: "vanilla"
        id: "IRON_INGOT"
        amount: 2
      - source: "vanilla"
        id: "STICK"
        amount: 1
    
    craft-time: 5.0
    success-rate: 100       # 必定成功
```

## 兼容性

- 默认值：`success-rate: 100`, `consume-on-fail: true`
- 现有配方无需修改，保持向后兼容
