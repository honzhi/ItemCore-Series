# 开发日志 - DEV-018 - 实时队列更新

**开发日期**: 2026-05-27  
**开发阶段**: Phase 18  
**开发状态**: ✅ 已完成

---

## 📋 开发目标

实现队列的实时更新机制，当玩家连续点击添加任务时，队列状态能够立即反映在 GUI 中，而不需要等待 1 秒的定时刷新。

---

## 🔧 问题描述

**用户反馈**: 其他插件可以实现立即实时更新队列的增加，而我们的插件如果是连续点击，队列是每秒更新。

**原因分析**:
- 原来的队列更新依赖于 Bukkit 的 `scheduleSyncRepeatingTask`
- 每 20 tick（1秒）更新一次
- 即使队列立即变化，也要等到下一个定时任务执行才会更新显示

**更新机制对比**:

| 机制 | 更新时机 | 延迟 |
|------|----------|------|
| 定时轮询（原） | 每 20 tick | 0-1 秒 |
| 事件驱动（新） | 队列变化时 | 几乎实时 |

---

## 🛠 实现方案

采用 **事件驱动 + 定时刷新** 结合的方式：

1. **事件驱动**: 队列变化时立即通知 GUI 更新
2. **定时刷新**: 用于同步剩余时间（每 1 秒）


## 📝 具体实现

### 1. 创建更新接口

**文件**: `gui/QueueUpdateable.java`

```java
package com.minemart.itemcoreforge.gui;

public interface QueueUpdateable {
    void updateQueueDisplay();
}
```

### 2. 在 enqueueCraft() 中触发事件

**文件**: `task/CraftScheduler.java`

```java
// 修复前
CraftTask task = new CraftTask(player, forge, recipe, 1);
queue.addTask(task);

if (queue.getCurrentTask() == task) {
    startCraftTask(player, task, queue);
}

// 修复后
CraftTask task = new CraftTask(player, forge, recipe, 1);
queue.addTask(task);

if (queue.getCurrentTask() == task) {
    startCraftTask(player, task, queue);
}

// 触发事件通知 GUI 更新
RecipeQueueEvent queueEvent = new RecipeQueueEvent(player, forge, queue);
Bukkit.getPluginManager().callEvent(queueEvent);
```

### 3. 在 BaseMenu.open() 中注册 GUI

**文件**: `gui/BaseMenu.java`

```java
// 修复前
public void open() {
    player.openInventory(inventory);
}

// 修复后
public void open() {
    plugin.getGuiListener().registerMenu(player, this);
    player.openInventory(inventory);
}
```

### 4. GUI 实现更新接口

**文件**: `gui/CustomForgeGUI.java`

```java
public class CustomForgeGUI extends BaseMenu implements QueueUpdateable {
    // ...
    // 已有 updateQueueDisplay() 方法
}
```

**文件**: `gui/QueueDisplayGUI.java`

```java
public class QueueDisplayGUI extends BaseMenu implements QueueUpdateable {
    // ...
    
    @Override
    public void updateQueueDisplay() {
        render();  // 调用现有的 render() 方法
    }
}
```

### 5. 在 GuiListener 中监听事件

**文件**: `gui/GuiListener.java`

```java
@EventHandler
public void onRecipeQueueUpdate(RecipeQueueEvent event) {
    Player player = event.getPlayer();
    BaseMenu menu = openMenus.get(player.getUniqueId());
    
    if (menu instanceof QueueUpdateable) {
        ((QueueUpdateable) menu).updateQueueDisplay();
    }
}
```

## 🎯 设计优势

1. **响应迅速**: 队列变化立即反映在 GUI 中
2. **性能良好**: 事件驱动避免了不必要的轮询
3. **向后兼容**: 保留定时刷新用于同步剩余时间
4. **解耦设计**: 使用事件机制，不需要 GUI 知道调度器的存在
5. **可扩展**: 其他插件也可以监听 RecipeQueueEvent
