# ItemCoreForge 开发方案

---

## 项目概述

### 项目名称
**ItemCoreForge** - ItemCore 物品锻造系统

### 定位
ItemCoreForge 是插件矩阵中基于 ItemCore 的核心拓展插件之一，提供**自定义锻造台系统**。通过配置锻造台和配方，让玩家可以使用 ItemCore 物品进行合成/制作。

### 设计理念
> **"通过锻造台，让物品创造成为可能"**
>
> - 以"锻造台"为单位组织配方
> - 支持 ItemCore 物品和原版物品混合使用
> - 引入**制作队列系统**，支持异步制作
> - 支持**自定义 GUI 布局**
> - 指令极简：`/icf <锻造台ID>` 和 `/icf reload`

### 与原版的对比

| 原版机制 | ItemCoreForge 对应 |
|----------|-------------------|
| 工作台 | 锻造台（type: crafting_table） |
| 熔炉 | 锻造台（type: furnace） |
| 酿造台 | 锻造台（type: brewing_stand） |
| 原版合成 | 锻造台（type: custom） |

### 锻造台类型

| 类型 | 说明 | 特殊功能 |
|------|------|----------|
| `crafting_table` | 工作台类型 | 复用原版工作台配方系统 |
| `furnace` | 熔炉类型 | 复用原版熔炉配方系统 |
| `brewing_stand` | 酿造台类型 | 复用原版酿造台配方系统 |
| `custom` | 自定义类型 | 完全自定义 GUI 布局和配方 |

## 核心功能设计

### 1. 锻造台系统

#### 设计理念
每个"锻造台"是一个独立的配方集合。玩家通过指令 `/icf <锻造台ID>` 打开该锻造台，查看配方并进行制作。支持**制作队列系统**，玩家可以同时排队制作多个物品。

#### 锻造台配置格式（v2.0）

```yaml
# plugins/ItemCoreForge/forges/铁匠铺.yml

forge-id: "铁匠铺"
display-name: "&6铁匠铺"

settings:
  max-queue-size: 6
  type: custom
  layout-file: "Default.yml"

recipes:
  配方名称1:
    output:
      source: "itemcore"
      category: "weapons"
      id: "iron_sword"
      amount: 1
      check-durability: false
    
    materials:
      - source: "itemcore"
        category: "misc"
        id: "iron_ingot"
        amount: 2
        check-durability: false
      - source: "vanilla"
        id: "STICK"
        amount: 1
        check-durability: false
    
    conditions:
      - type: "level"
        value: 5
    
    craft-time: 3.5

  配方名称2:
    output:
      source: "itemcore"
      category: "weapons"
      id: "flame_blade"
      amount: 1
      check-durability: false
    
    materials:
      - source: "itemcore"
        category: "misc"
        id: "fire_essence"
        amount: 3
        check-durability: false
    
    conditions:
      - type: "level"
        value: 15
    
    craft-time: 5.0
```

#### 物品引用格式

**简化字符串格式**（适合简单配置）：
```yaml
output: "itemcore:weapons/iron_sword*1"
materials:
  - "itemcore:misc/iron_ingot*2"
  - "vanilla:STICK*1"
```

**对象格式**（适合复杂配置）：
```yaml
output:
  source: "itemcore"  # itemcore | vanilla
  category: "weapons"  # ItemCore 物品分类（仅 itemcore 需要）
  id: "iron_sword"
  amount: 1
  check-durability: false  # 是否检查耐久度
```

#### 条件配置格式

```yaml
conditions:
  - type: "level"           # 等级条件
    value: 15
  - type: "permission"       # 权限条件
    node: "forge.craft.legendary"
  - type: "money"            # 经济条件（需要 Vault）
    amount: 1000
```


### 2. 指令系统

### 指令总览

| 指令 | 别名 | 权限 | 说明 |
|------|------|------|------|
| `/icf <锻造台名称>` | `/itemcoreforge <锻造台名称>` | `itemcoreforge.forge.<名称>` | 打开指定锻造台 |
| `/icf reload` | `/itemcoreforge reload` | `itemcoreforge.reload` | 重载所有锻造台配置 |

### 指令详情

#### 1. 打开锻造台

```
/icf <锻造台名称>
/icf 武器锻造台
/icf 护甲锻造台
/icf 药水酿造台
```

- **权限**: `itemcoreforge.forge.<锻造台名称>`（如 `itemcoreforge.forge.武器锻造台`）
- **功能**: 打开指定锻造台的 GUI，显示该锻造台的所有配方
- **显示**: 配方以"结果物品"的形式展示在 GUI 中

#### 2. 重载配置

```
/icf reload
```

- **权限**: `itemcoreforge.reload`
- **功能**: 重新加载所有锻造台配置文件
- **适用**: 修改配方后无需重启服务器

### 3. 权限系统

| 权限节点 | 说明 | 默认拥有者 |
|----------|------|-----------|
| `itemcoreforge.forge.<锻造台名称>` | 访问指定锻造台 | OP |
| `itemcoreforge.craft.<锻造台名称>` | 在指定锻造台制作（可选，可与访问权限合并） | OP |
| `itemcoreforge.reload` | 重载配置 | OP |
| `itemcoreforge.admin` | 管理员权限（包含所有） | OP |

#### 权限示例

```yaml
# ItemCoreForge 的 plugin.yml 中将自动生成
permissions:
  itemcoreforge.admin:
    description: Full admin access
    default: op
    children:
      itemcoreforge.reload: true
      # 锻造台权限动态生成
      itemcoreforge.forge.武器锻造台: true
      itemcoreforge.forge.护甲锻造台: true
      ...
```

#### 制作流程

1. 玩家打开锻造台（`/icf 武器锻造台`）
2. 点击要制作的物品（如传说之剑）
3. 打开配方详情界面
4. 点击"开始锻造"按钮
5. 系统检查：
   - 材料是否足够
   - 条件是否满足（等级、权限）
   - 是否在冷却中
6. 消耗材料，给予结果（或失败）

### 6. GUI 布局系统（自定义锻造台）

#### 布局文件格式

```yaml
# plugins/ItemCoreForge/layouts/Default.yml

settings:
  rows: 6

layout:
  - 'AAAAAAAAA'  # 第一行：边框
  - 'AAA0E0AAA'  # 第二行：材料槽 + 输出槽
  - 'AZ000000A'  # 第三行：确认按钮 + 材料槽
  - 'AAA0X0AAA'  # 第四行：材料槽 + 队列显示
  - 'AAAAAAAAA'  # 第五行：边框
  - 'AXXXXXXXA'  # 第六行：队列信息

slots:
  A:
    material: BLACK_STAINED_GLASS_PANE
    display-name: ""
    function: border

  0:
    material: AIR
    function: material_slot
    show-item: true

  E:
    material: AIR
    function: output_slot

  X:
    material: GREEN_STAINED_GLASS_PANE
    display-name: "&a制作队列"
    function: queue_display

  Z:
    material: LIME_WOOL
    display-name: "&a开始制作"
    function: confirm_button
    lore:
      - "&7点击开始制作"
      - "&7制作时间: {craft_time}秒"
      - ""
      - "{conditions_met}&7条件: &a已满足"
      - "{conditions_not_met}&7条件: &c未满足"
```

#### 功能选项说明

| Function | 说明 | 参数 |
|----------|------|------|
| `border` | 装饰边框 | 无 |
| `material_slot` | 材料槽位 | `show-item`: 是否显示物品 |
| `output_slot` | 输出物品显示 | 无 |
| `queue_display` | 制作队列显示（显示剩余时间） | 无 |
| `confirm_button` | 确认制作按钮 | `lore` 支持条件占位符 |
| `cancel_button` | 取消按钮 | 无 |

#### 队列显示说明

`queue_display` 功能槽位用于显示当前制作队列中的物品：
- 显示正在制作的物品图标
- **在物品 Lore 中显示剩余制作时间**
- 显示队列中等待的物品数量

**队列物品 Lore 示例**：
```
&a正在制作: 传说之剑
&7剩余时间: &e5.2秒
&7队列中还有: &e2 &7个物品
```

#### 条件占位符

在按钮的 lore 中可以使用以下占位符：

- `{craft_time}` - 制作时间
- `{material_count}` - 材料数量
- `{level_required}` - 所需等级
- `{conditions_met}` - 已满足的条件（绿色）
- `{conditions_not_met}` - 未满足的条件（红色）

## 与 ItemCore 的集成

### 使用的 ItemCore API

```java
// 验证物品是否存在
ItemCoreAPI.hasItem(String itemId)

// 获取物品实例（用于检查玩家是否拥有）
ItemCoreAPI.getItemStack(String itemId)
ItemCoreAPI.getItemStack(String itemId, int amount)

// 获取物品定义
ItemCoreAPI.getCustomItem(String itemId)

// 给予玩家物品
ItemCoreAPI.giveItem(Player player, String itemId, int amount)
```

**当前 ItemCore API 已完全满足需求！**

### 异步制作流程

```java
public class CraftScheduler {

    // 1. 加入制作队列
    public boolean enqueue(Player player, Recipe recipe, int amount) {
        // 检查队列是否已满
        if (playerQueue.getQueueSize() >= maxQueueSize) {
            return false;
        }

        // 检查材料
        if (!conditionChecker.hasMaterials(player, recipe)) {
            return false;
        }

        // 检查条件
        if (!conditionChecker.checkConditions(player, recipe)) {
            return false;
        }

        // 消耗材料
        conditionChecker.consumeMaterials(player, recipe);

        // 创建制作任务
        CraftTask task = new CraftTask(player, recipe, amount);
        task.setStartTime(System.currentTimeMillis());
        task.setDuration(recipe.getCraftTime() * 1000L);

        // 加入队列
        playerQueue.addTask(task);

        // 启动异步任务
        startAsyncCraft(task);

        return true;
    }

    // 2. 异步执行制作
    private void startAsyncCraft(CraftTask task) {
        Bukkit.getScheduler().runTaskLater(plugin, () -> {
            // 检查是否被取消
            if (task.isCancelled()) {
                return;
            }

            // 执行制作
            executeCraft(task);
        }, (long) (task.getDuration() / 50));  // 转换为tick
    }

    // 3. 完成制作
    private void executeCraft(CraftTask task) {
        Player player = task.getPlayer();
        Recipe recipe = task.getRecipe();

        // 触发事件
        RecipePreCraftEvent event = new RecipePreCraftEvent(player, recipe);
        Bukkit.getPluginManager().callEvent(event);

        if (event.isCancelled()) {
            return;
        }

        // 给予物品
        ItemCoreAPI.giveItem(player,
            recipe.getOutput().getId(),
            recipe.getOutput().getAmount()
        );

        // 触发完成事件
        RecipeCraftedEvent craftedEvent = new RecipeCraftedEvent(player, recipe, true);
        Bukkit.getPluginManager().callEvent(craftedEvent);

        // 触发队列事件
        RecipeQueueEvent queueEvent = new RecipeQueueEvent(player, recipe);
        Bukkit.getPluginManager().callEvent(queueEvent);
    }
}
```

### 材料检查（支持 ItemCore + 原版）

```java
public class MaterialChecker {

    public boolean hasMaterials(Player player, Recipe recipe) {
        for (ItemReference material : recipe.getMaterials()) {
            ItemStack item = resolveItem(material);

            if (material.isItemCore()) {
                // ItemCore 物品：使用 containsAtLeast 检查
                if (!player.getInventory().containsAtLeast(item, material.getAmount())) {
                    return false;
                }
            } else {
                // 原版物品：直接检查
                if (!player.getInventory().containsAtLeast(item, material.getAmount())) {
                    return false;
                }
            }
        }
        return true;
    }

    public void consumeMaterials(Player player, Recipe recipe) {
        for (ItemReference material : recipe.getMaterials()) {
            ItemStack item = resolveItem(material);

            if (material.isCheckDurability()) {
                // 检查耐久度
                consumeWithDurability(player, item, material.getAmount());
            } else {
                // 不检查耐久度
                player.getInventory().removeItem(item.clone().setAmount(material.getAmount()));
            }
        }
    }

    private ItemStack resolveItem(ItemReference ref) {
        if (ref.isItemCore()) {
            // 从 ItemCore 获取物品
            return ItemCoreAPI.getItemStack(ref.getId(), ref.getAmount());
        } else {
            // 原版物品
            Material material = Material.valueOf(ref.getId().toUpperCase());
            return new ItemStack(material, ref.getAmount());
        }
    }
}
```

## API 事件系统

### 事件列表

| 事件 | 说明 | 可取消 |
|------|------|--------|
| `RecipePreCraftEvent` | 制作前事件 | ✅ |
| `RecipeCraftedEvent` | 制作完成事件 | ❌ |
| `RecipeQueueEvent` | 队列更新事件 | ❌ |

### 事件使用示例

```java
// 监听制作前事件（可取消）
@EventHandler
public void onPreCraft(RecipePreCraftEvent event) {
    Player player = event.getPlayer();
    Recipe recipe = event.getRecipe();

    // 检查是否有特殊权限
    if (player.hasPermission("special.bypass")) {
        // 降低制作时间
        event.setCraftTime(recipe.getCraftTime() * 0.5);
    }
}

// 监听制作完成事件
@EventHandler
public void onCrafted(RecipeCraftedEvent event) {
    Player player = event.getPlayer();
    Recipe recipe = event.getRecipe();

    // 播放完成音效
    player.playSound(player.getLocation(), Sound.ENTITY_PLAYER_LEVELUP, 1.0f, 1.0f);

    // 发送公告（如果是传说物品）
    if (recipe.getOutput().getId().contains("legendary")) {
        Bukkit.broadcastMessage(ChatColor.GOLD + player.getName() + " 制作出了传说物品！");
    }
}
```
