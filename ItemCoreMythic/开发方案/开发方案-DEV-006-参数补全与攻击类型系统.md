## 一、attackType — 攻击类型系统 ⭐ 新增

### 1.1 需求

当前 ICDamage 把所有伤害视为"技能伤害"，无法区分：

```
普攻触发技能（应享受暴击率、吸血）
  vs
技能释放（独立暴击，无吸血）
```

新增 `attackType` 参数，两个值：

| 值 | 行为 |
|----|------|
| `attack` | 视为**普攻**，享受玩家攻击属性（暴击率、吸血、攻击加成），`crit=false` 时例外 |
| `skill` | 视为**技能**（默认），使用法术属性，不触发吸血 |

### 1.2 参数设计

```yaml
Skills:
  - icdamage{
      amount="<ic.attack_damage>*1.5";
      type=physical;
      attackType=attack;   # 新增：此次触发视为普攻
      element=fire;
      crit=true
    } @EIR{r=5}
```

| 参数 | 必填 | 默认值 | 说明 |
|------|:---:|--------|------|
| `attackType` | ✗ | `skill` | `attack`（普攻）/ `skill`（技能） |

### 1.3 行为差异表

| 维度 | `attack` | `skill`（默认） |
|------|----------|-----------------|
| 伤害加成 | `attack_damage` + `physical_damage`% | `spell_power` / `adaptive_force` |
| 暴击率来源 | 玩家 `crit_chance` 属性 | 任务 `spell_crit_chance`（若有），否则无暴击 |
| 暴击伤害 | 玩家 `crit_damage` 属性 | 同左（或独立 `spell_crit_damage`） |
| 吸血 | ✅ 享受玩家吸血属性 | ❌ 不触发 |
| 伤害类型 | **强制 PHYSICAL** | 按 `type` 参数指定 |
| `crit=false` | 关闭暴击，吸血不受影响 | 关闭暴击 |

### 1.4 ICDamageMechanic 改动

```java
private final AttackType attackType;  // 新增枚举

public ICDamageMechanic(MythicLineConfig config) {
    String atStr = config.getString(new String[]{"attackType", "at"}, "skill");
    this.attackType = "attack".equalsIgnoreCase(atStr) ? AttackType.ATTACK : AttackType.SKILL;
    // ...
}

@Override
public SkillResult castAtEntity(SkillMetadata data, AbstractEntity target) {
    // ...
    LivingEntity attacker = getCaster(data);

    if (attackType == AttackType.ATTACK) {
        // 普攻模式：获取玩家攻击属性
        if (attacker instanceof Player player) {
            AttributeContainer attrs = AttributeCalculator.calculatePlayerAttributes(player);
            critChance = attrs.getAttribute(CustomAttribute.CRIT_CHANCE);
            critDamage = attrs.getAttribute(CustomAttribute.CRIT_DAMAGE);
            lifesteal = attrs.getAttribute(CustomAttribute.LIFESTEAL);  // 待ItemCore实现
        }
        damageType = DamageTag.PHYSICAL;  // 强制物理
    }

    DamageRequest request = DamageRequest.builder()
        .attackType(attackType)     // 新增
        .critChance(critChance)     // 当前为默认0，需传入
        .critDamage(critDamage)     // 当前为默认150，需传入
        .lifesteal(lifesteal)       // 新增，待ItemCore实现
        .build();
}
```

### 1.5 ItemCore 侧需求

| 改动 | 说明 |
|------|------|
| `DamageRequest.attackType` | 新字段，枚举 {ATTACK, SKILL} |
| `DamageRequest.lifesteal` | 新字段，吸血比例（0.0~1.0） |
| `DamageManager` 吸血逻辑 | 根据 `attackType` + `lifesteal` 计算回血 |
| `DamageManager` 暴击来源区分 | `attackType=attack` 用 `critChance`；`skill` 走独立法术暴击 |

### 1.6 枚举定义

```java
// ItemCore — 放在 DamageRequest 或独立文件
public enum AttackType {
    ATTACK,   // 普攻触发
    SKILL     // 技能释放
}
```

## 三、DamageManager 改造清单

当前 DamageManager 只用了 `baseDamage`、`damageType`、`canCrit` 三个字段。需补齐：

```
DamageRequest 字段                 当前状态      目标
────────────────────────────────────────────────────────
.attacker                            ✅ 已用
.victim                              ✅ 已用
.baseDamage                          ✅ 已用
.damageType                          ✅ 已用
.element                             ❌ 忽略     → 元素加成计算
.canCrit                             ✅ 已用（但critChance=0永远不暴）
.critChance                          ❌ 未传入   → 从攻击者属性读取
.critDamage                          ❌ 未传入   → 从攻击者属性读取
.penetration                         ❌ 忽略     → 扣减目标抗性
.castId                              ❌ 未用     → SkillMetadataManager
.attackType  (新增)                   —         → 区分普攻/技能
.lifesteal   (新增)                   —         → 吸血回血
```
