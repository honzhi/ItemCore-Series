# 开发日志 DEV-001 - 项目初始化与核心实现

## 概述

本次开发完成 ItemCoreMythic 桥接插件的核心功能实现，包括项目初始化、Provider 架构、Mechanic、Condition、Placeholder、监听器等所有核心组件。

## 开发内容

### 1. 项目初始化

**文件**:
- `pom.xml` - Maven 项目配置
- `src/main/resources/plugin.yml` - 插件配置
- `src/main/java/com/minemart/itemcoremythic/ItemCoreMythic.java` - 主类

**功能**:
- 配置 Maven 依赖（Bukkit、ItemCore、MythicMobs）
- 配置插件元数据（名称、版本、依赖项）
- 实现主类 `onEnable()` 和 `onDisable()` 生命周期
- 初始化组件、注册监听器和 Mechanic

### 2. Provider 架构

**文件**:
- `bridge/SkillBridge.java` - 技能桥接器
- `provider/SkillProvider.java` - 技能提供者接口
- `provider/impl/MythicMobsSkillProvider.java` - MythicMobs 实现

**功能**:
- `SkillBridge` 作为中央路由器，根据 `provider` 字段分发技能
- `MythicMobsSkillProvider` 通过 `SkillExecutor` 执行 MM 技能
- 支持未来扩展其他技能系统（MMOItems/MagicSpells）

### 3. Mechanic 实现

**文件**:
- `mechanic/ICDamageMechanic.java` - 伤害机制
- `mechanic/ICHealMechanic.java` - 治疗机制

**功能**:
- 继承 `ITargetedEntitySkill` 标准接口
- 使用 MM `PlaceholderString` 解析表达式
- 支持 `amount`、`type`、`element`、`crit`、`penetration` 参数
- 通过 `ItemCoreAPI.processDamage()` / `processHeal()` 委托计算

### 4. Placeholder 实现

**文件**:
- `placeholder/ICPlaceholderExpansion.java` - 占位符扩展

**功能**:
- 启动时遍历 `CustomAttribute.values()` 自动注册
- 占位符格式：`ic.{attribute_config_key}`（如 `ic.attack_damage`）
- ItemCore 新增属性后 MM 自动支持，无需修改代码

### 5. Condition 实现

**文件**:
- `condition/ICAttributeCondition.java` - 属性条件

**功能**:
- 单一通用 Condition 覆盖所有属性
- 参数：`attr`（属性名）、`op`（操作符）、`value`（值）
- 支持操作符：`==`、`!=`、`>`、`<`、`>=`、`<=`

### 6. 监听器实现

**文件**:
- `listener/SkillTriggerListener.java` - 技能触发监听
- `listener/MMLoadListener.java` - MM 重载监听

**功能**:
- 监听 `ItemSkillTriggerEvent`，通过 SkillBridge 执行技能
- 支持目标实体和位置传递给 MM 技能

## 依赖关系

```
ItemCoreMythic (桥接层)
  ├─ ItemCore (依赖)
  │  ├─ ItemSkillTriggerEvent
  │  ├─ ItemCoreAPI.processDamage()
  │  └─ ItemCoreAPI.processHeal()
  └─ MythicMobs (依赖)
     ├─ SkillMechanic
     ├─ ITargetedEntitySkill
     ├─ PlaceholderManager
     └─ SkillExecutor
```

## 架构设计原则

1. **职责单一**：每个 Provider/Mechanic/Condition 仅一个功能
2. **接口隔离**：仅依赖 ItemCoreAPI，不依赖内部实现
3. **动态扩展**：Placeholder/Condition 自动适配 ItemCore 属性
4. **Provider 路由**：支持多技能系统，避免 ItemCore 与 MM 绑定
5. **事件驱动**：通过 ItemSkillTriggerEvent 解耦

## 使用示例

### MythicMobs 技能配置

```yaml
Skills:
  - icdamage{
      amount="<ic.attack_damage>*2 + <ic.spell_damage>*1.5",
      type=spell,
      element=fire,
      crit=true
    } @EIR{r=5}
  
  - icheal{
      amount="<ic.spell_power>"
    } @P{self}

Conditions:
  - icAttribute{attr=attack_damage;op=>=;value=50}
```

### 物品技能配置

```yaml
skills:
  on_hit:
    provider: mythicmobs
    skill: fire_slash
```
