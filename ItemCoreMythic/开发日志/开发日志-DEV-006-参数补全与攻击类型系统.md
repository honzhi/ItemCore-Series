## 1. attackType 系统

`icdamage` 新增 `attackType` 参数：

| 值 | 行为 |
|----|------|
| `attack` | 普攻模式 — 强制 PHYSICAL，使用玩家 CRIT_CHANCE/CRIT_DAMAGE，支持吸血 |
| `skill`（默认） | 技能模式 — 走传入的 type/暴击参数 |

**ItemCore 侧：**
- 新增 `AttackType` 枚举（ATTACK / SKILL）
- `DamageRequest` 新增 `attackType`、`lifesteal` 字段
- `DamageManager.processDamage()` 根据 attackType 分流：
  - ATTACK：自动加 ATTACK_DAMAGE + PHYSICAL_DAMAGE%，使用玩家暴击属性
  - SKILL：使用传入的 damageType 和暴击参数

## 2. 元素系统对接

`element` 参数支持中英双语：

| 配置 | 对应元素 |
|------|----------|
| `fire` / `liuhuo` | LIUHUO（流火） |
| `ice` / `hanshuang` | HANSHUANG（寒霜） |
| `thunder` / `leizhe` | LEIZHE（雷蛰） |

DamageManager 根据 ElementMastery 和 ElementResistance 计算元素倍率。

## 3. 参数补全

- **穿透**：技能 `penetration` 参数 + 玩家属性穿透叠加
- **吸血**：`lifesteal` 参数（仅 attackType=attack 生效）
- **暴击**：attackType=attack 时自动读取玩家暴击属性

## 4. MM 参数解析 Bug 修复

**问题**：MythicMobs 的 MythicLineConfig 无法正确解析带引号的参数。
```
icdamage{amount="<ic.spell_power>",type=physical}
```
`config.getString("amount")` 返回 `"<ic.spell_power>",type=physical`（整个后半段），导致 `config.getString("type")` 找不到 key，返回默认值 `spell`。

**修复**：从 `rawAmount` 字符串尾部手动提取所有参数（type、element、crit、penetration、attackType、lifesteal），不再依赖 MM 的参数解析器。

## 5. 伤害类型默认值

`type` 默认值从 `physical` 改为 `spell`（法杖为主，法术伤害更常用）。

## 测试要点

1. 无参数：默认 type=spell，显示 ✨ 青色
2. type=physical：显示 ⚔ 灰色
3. element=liuhuo：显示 🔥 红色（覆盖类型）
4. type=spell,element=hanshuang：显示 ❄ 蓝色
5. attackType=attack：强制 PHYSICAL，走玩家暴击属性
