# DEV-004: 技能执行链路修复 - 使用 BukkitAPIHelper

**日期**: 2026-06-02
**状态**: ✅ 已完成

## 问题描述

联调时 MythicMobs 找到了技能但执行失败，日志显示:
```
[MythicMobs] Couldn't execute skill 'text': Enable debugging for a stack trace.
[ItemCoreMythic] Failed to get SkillCaster: io.lumine.mythic.api.skills.SkillCaster.of(io.lumine.mythic.api.adapters.AbstractEntity)
```

## 根因分析

之前的实现尝试构造 `SkillMetadata` → `skill.execute(metadata)` 的标准 API 链路，但以下几个方法在 MythicMobs 5.11.0 中不存在：

1. `SkillCaster.of(AbstractEntity)` — **不存在**。`SkillCaster` 是接口，没有静态工厂方法
2. `SkillTrigger.COMMAND` 等静态常量 — **不存在**。需通过 `SkillTrigger.get("API")` 获取

## 解决方案

通过反编译 MythicMobs 5.11.0 jar 发现官方提供了 **`BukkitAPIHelper`**：

```java
// io.lumine.mythic.bukkit.BukkitAPIHelper
public boolean castSkill(Entity, String)                                    // 无目标
public boolean castSkill(Entity, String, Location)                          // 指定位置
public boolean castSkill(Entity, String, Entity, Location, Collection, Collection, float)  // 完整版
```

通过 `MythicBukkit.inst().getAPIHelper()` 获取实例即可。

## 修改内容

### MythicMobsSkillProvider.java
- 完全重写，从 275 行缩减到 63 行
- 移除所有反射逻辑（SkillCaster、SkillMetadata、SkillTrigger）
- 改用 `BukkitAPIHelper.castSkill()` 直接执行
- 支持 target 非空时使用完整版 castSkill

### SkillBridge.java
- 构造函数调用从 `new MythicMobsSkillProvider(plugin, mythicMobs)` 改为 `new MythicMobsSkillProvider(plugin)`

## 经验教训

1. **不要猜测第三方 API**。先用 `javap` 反编译查看实际方法签名，再决定调用方式
2. **优先使用官方提供的简单 API**。`BukkitAPIHelper` 比构造 `SkillMetadata` 链路简单得多
3. **反射是最后手段**，不是首选方案。当有公开 API 时直接用
