## 关键设计决策

### 1. 配置文件结构

```yaml
damage_indicators:
  types:
    physical:
      format: '{icon} &f{value}'
      icon: '⚔'
      color: '&7'
    
    spell:
      format: '{icon} &f{value}'
      icon: '✨'
      color: '&b'
    
    crit:
      format: '{icon} &f&l{value}'
      icon: '⚡'
      color: '&c'
      show_indicator: true
      indicator: '暴击!'
      indicator_color: '&e'
```

### 2. 伤害类型判断逻辑

```java
private String getDamageType(ItemCoreDamageEvent event) {
    if (event.isSpellDamage()) {
        return "spell";
    }
    return "physical";
}
```

### 3. 显示优先级

- 暴击会覆盖物理/法术的显示格式
- 暴击时额外显示暴击提示文字（如"暴击!"）

### 4. 自定义字体支持

每种伤害类型都有独立的字体映射配置：
- `custom_font.physical` - 物理伤害字体
- `custom_font.spell` - 法术伤害字体
- `custom_font.crit` - 暴击伤害字体
