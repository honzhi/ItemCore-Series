## 关键设计决策

### 1. 伤害显示方案

采用 **ArmorStand 悬浮文字** 方案：
- 在目标实体头顶生成不可见的盔甲架
- 通过 `setCustomName()` 显示伤害数值
- 带有平滑的上升动画效果
- 自动在设定时间后移除

### 2. 配置系统设计

配置文件结构：
```yaml
damage_indicators:
  enabled: true              # 是否启用
  min_damage: 0.1            # 最小伤害阈值
  format: '{icon} &f{value}' # 普通伤害格式
  crit_format: '{icon} &f&l{value}' # 暴击伤害格式（加粗）
  decimal_format: '0.#'      # 小数格式
  
  animation:
    duration-ticks: 40       # 显示时长
    rise-height: 1.5         # 上升高度
    vertical-offset: 0.5     # 初始偏移
  
  icons:
    normal: '❤'              # 普通伤害图标
    crit: '⚡'                # 暴击伤害图标
  
  colors:
    normal: '&7'             # 普通伤害颜色
    crit: '&c'               # 暴击伤害颜色
    custom_font:             # 自定义字体配置
      enabled: false
      normal: { ... }
      crit: { ... }
```

### 3. 命令系统

| 命令 | 说明 | 权限 |
|------|------|------|
| `/icrpg` | 显示帮助信息 | `itemcorerpg.command.help` |
| `/icrpg reload` | 重载配置文件 | `itemcorerpg.command.reload` |
| `/icrpg help` | 显示帮助信息 | `itemcorerpg.command.help` |

支持 Tab 补齐功能。

### 4. 事件监听方案

最初监听 `EntityDamageByEntityEvent`，但存在暴击检测不准确的问题。

最终改为监听 ItemCore 的自定义事件 `ItemCoreDamageEvent`：
- 直接获取准确的伤害数值
- 直接获取准确的暴击状态
- 避免重复计算
