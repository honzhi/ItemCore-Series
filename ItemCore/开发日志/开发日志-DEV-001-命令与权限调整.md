## 关键设计决策

### 1. 命令删除策略
- 保留核心功能命令（help、open、give、reload）
- 移除冗余或较少使用的命令（list、info）
- 通过 GUI 提供查看物品信息的功能（Shift+点击）

### 2. DEBUG 日志控制
- 添加 `debugMode` 变量检查所有 DEBUG 日志输出
- 只有当 `config.yml` 中的 `debug-mode: true` 时才输出
- 保持日志前缀 `[DEBUG]` 不变，便于识别

### 3. 权限安全考虑
- 将 `/ic open` 命令默认权限改为 OP，防止普通玩家滥用
- 保留物品获取权限 `itemcore.gui.obtain` 为 OP 级别
- `itemcore.admin` 仍然包含所有子权限
