## 1. 项目概述

### 1.1 定位

ItemCoreLevel 是 ItemCore 生态下的一个扩展插件，独立运行，通过 `depend: [ItemCore]` 与核心插件建立依赖关系。它不依赖原版 Minecraft 的经验值系统，而是构建一套独立的玩家等级体系。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| 玩家等级 | 独立的玩家等级体系，不依赖原版经验 |
| 经验获取 | 通过 MythicMobs 怪物掉落经验值（可配置） |
| 等级属性奖励 | 每级可配置各种 ItemCore 属性，通过 AttributeProvider 集成 |
| 数据存储 | MySQL 持久化存储玩家等级和进度 |
| 经验提示 | 获取经验时聊天栏显示，支持可配置的格式和颜色 |
| 升级提示 | 3 种模式：聊天栏 / ActionBar / Title，可配置 |
| PlaceholderAPI | 提供等级/经验相关变量供其他插件使用 |
| 管理指令 | 查询/设置/重置玩家等级和经验 |

### 1.3 与 ItemCore 生态的关系

```
ItemCore (核心)
  ├── ItemCoreRPG (伤害显示)
  ├── ItemCoreForge (锻造系统)
  ├── ItemCoreMythic (MythicMobs 技能桥接)
  ├── ItemCoreTrinkets (饰品系统)
  └── ItemCoreLevel (玩家等级)  ← 本插件
       ├── 依赖 ItemCore (提供属性计算管线)
       ├── 软依赖 MythicMobs (作为经验掉落源)
       └── 自身独立运行 (独立存储、独立配置)
```

### 2.2 数据流

```
MythicMobs 怪物死亡
       │
       ▼
MythicExpListener (监听 MythicMobs 死亡事件)
       │
       ▼
读取 monster-exp 配置中该怪物的经验值
       │
       ▼
ExpManager.addExp(player, amount)
       │
       ├── 触发 PlayerGainExpEvent (可取消)
       │
       ├── 发送获取经验提示（聊天栏，格式见 config）
       │
       ▼
LevelManager.checkLevelUp(player)
       │
       ├── 计算当前经验是否达到升级阈值
       │
       ├── 否 → 结束
       │
       └── 是 → 执行升级
                   │
                   ├── 更新玩家等级数据
                   ├── 触发 PlayerLevelUpEvent
                   ├── 发送升级提示（按配置模式：聊天/ActionBar/Title）
                   ├── LevelAttributeProvider 重新计算属性
                   └── ItemCoreAPI.refreshPlayerAttributes(player)
```

## 3. 模块划分

### 3.1 主类 (ItemCoreLevel.java)

- 继承 `JavaPlugin`
- `onEnable` 阶段:
  1. 检查依赖 (ItemCore)
  2. 保存默认 config.yml
  3. 加载配置
  4. 初始化 MySQL 数据源
  5. 注册监听器（PlayerListener）
  6. 若 MythicMobs 存在，注册 MythicExpListener
  7. 注册指令
  8. 注册 PlaceholderAPI 变量
  9. 注册 AttributeProvider 到 ItemCoreAPI
  10. 加载在线玩家数据
- `onDisable` 阶段:
  1. 保存所有在线玩家数据
  2. 关闭 HikariCP 连接池

### 3.2 配置文件模块

#### ConfigManager.java
- 加载 config.yml
- 提供所有配置字段的 getter，按区域划分方法

| 配置区域 | 对应方法 |
|---------|---------|
| 基础设置 | `isDebug()`, `getStartLevel()`, `getStartExp()`, `getMaxLevel()` |
| 等级曲线 | `getLevelCurveMode()`, `getLevelCurveExpression()`, `getLevelCurveList()` |
| 通用属性奖励 | `getGeneralRewards()` → `Map<CustomAttribute, Double>` |
| 特定等级奖励 | `getSpecificRewards()` → `Map<Integer, Map<CustomAttribute, Double>>` |
| 怪物经验 | `getMonsterExpDefault()`, `getMonsterExpMobs()` → `Map<String, Integer>`, `isUseMobLevel()`, `getMobLevelFormula()` |
| 经验来源 | `isMythicMobsExpEnabled()`, `isKillerOnly()`, `getExpMultiplier()` |
| 经验提示 | `getExpGainMessage()` |
| 升级提示 | `getLevelUpMode()`, `getLevelUpChatMessage()`, `getLevelUpActionBar()`, `getLevelUpTitle()`, `getLevelUpSubtitle()` |
| 数据库 | `getDatabaseHost()`, `getDatabasePort()`, ... |
| Placeholder | `getProgressBarLength()`, `getProgressBarDoneChar()`, ... |

### 3.3 监听器模块

#### MythicExpListener.java
- 仅在 MythicMobs 存在时注册
- 监听 `MythicMobDeathEvent`
- 计算经验值（配置映射 → 等级公式 → 默认值）
- 分配经验（击杀者/范围分享）
- 调用 `ExpManager.addExp()`

#### PlayerListener.java
- 玩家加入 → 异步 MySQL 加载 → 缓存到 LevelManager
- 玩家退出 → 保存到 MySQL → 清除缓存

### 3.4 核心管理器模块

#### LevelManager.java
- 内存缓存 `Map<UUID, PlayerData>`
- 等级计算：给定经验值计算应处等级
- 升级判定和执行
- 获取/设置玩家等级、经验

#### ExpManager.java
- 经验增减操作
- 调用 LevelManager 检查升级
- 触发事件和消息发送

### 3.5 存储模块

#### DataStorage.java (接口)

#### MySQLStorage.java
- HikariCP 连接池
- 单表 `ic_level_player_data`
- 异步写入（BukkitRunnable + 事务批量）

### 3.6 API 模块

#### LevelAPI.java
```java

    static int getLevel(Player player);

    static double getExpProgress(Player player);
    static void setLevel(Player player, int level);
    static void addExp(Player player, int amount, ExpSource source);
    static void setExp(Player player, int amount);
    static void resetPlayer(Player player);

```

#### PlayerLevelUpEvent.java
```java
public class PlayerLevelUpEvent extends Event implements Cancellable {
    Player getPlayer();
    int getOldLevel();
    int getNewLevel();
}
```

#### PlayerGainExpEvent.java
```java
public class PlayerGainExpEvent extends Event implements Cancellable {
    Player getPlayer();
    int getAmount();
    ExpSource getSource();
    void setAmount(int amount);
}

enum ExpSource {
    MOB_KILL,
    COMMAND,
    API
}
```

### 3.7 属性提供者模块

#### LevelAttributeProvider.java
- 实现 `AttributeProvider` 接口
- 根据玩家当前等级汇总属性
- `general` 模式：每级 × 等级
- `specific` 模式：达到等级即给（不重复叠加）
- 通过 `ItemCoreAPI.registerAttributeProvider()` 注册

### 3.8 变量模块

#### LevelExpansion.java
- 注册 PlaceholderAPI 扩展
- 提供 %itemcorelevel_*% 变量

### 3.9 工具模块

#### MessageUtil.java
- 颜色代码转换（`&` → `§`）
- 占位符替换（`{player}`, `{level}`, `{exp}`, `{amount}` 等）
- 发送方式：`sendChat()`, `sendActionBar()`, `sendTitle()`

## 4. 配置文件设计（单文件 config.yml）

所有配置项集中在 `config.yml` 中，随插件默认打包，首次运行自动生成。每段配置都有详细注释说明。

```yaml
# ========================================
# ItemCoreLevel 配置文件
# 基于 ItemCore 的玩家等级系统扩展插件
# ========================================

# ─────────────────────────────────────
# 调试模式
# ─────────────────────────────────────
# 启用后输出更详细的日志信息，用于问题排查
# 生产环境建议关闭
debug: false

# ─────────────────────────────────────
# 基础等级设置
# ─────────────────────────────────────

# 新玩家进入服务器时的初始等级
start-level: 1

# 新玩家进入服务器时的初始经验值
start-exp: 0

# 玩家可达到的最高等级
# 设为 0 表示无限制
max-level: 100

# ─────────────────────────────────────
# 等级经验曲线
# ─────────────────────────────────────
# 控制每升一级需要多少经验值
#
# mode: 经验计算模式
#   formula - 公式模式，使用 expression 字段的公式计算
#   list    - 列表模式，使用 list 字段手动指定每级所需经验
#
# expression: 公式表达式（仅 formula 模式有效）
#   可用变量: {level} = 当前等级
#   支持运算符: + - * / ^ ( )
#   示例:
#     100*{level}             → 线性增长，1级需100，2级需200
#     100*{level}^2           → 平方增长，1级需100，2级需400
#     100*{level}^2+50*{level} → 混合增长，1级需150，2级需500
#
# list: 经验列表（仅 list 模式有效）
#   第 0 项 = 1级升2级所需经验
#   第 1 项 = 2级升3级所需经验，以此类推
level-curve:
  mode: formula
  expression: "100*{level}^2+50*{level}"
  # list:
  #   - 100
  #   - 200
  #   - 400
  #   - 800

# ─────────────────────────────────────
# 等级属性奖励
# ─────────────────────────────────────
# 玩家升级后可获得的 ItemCore 属性加成
# 这些属性会通过 AttributeProvider 注册到 ItemCore 的属性计算系统
#
# 可用属性键（来自 ItemCore 的 CustomAttribute 枚举）:
#   attack_damage      攻击伤害        attack_speed       攻击速度
#   attack_range       攻击范围        health             生命值
#   movement_speed     移动速度        regeneration        生命恢复
#   knockback          击退             luck              幸运值
#   spell_damage       法术加成         physical_damage    物理加成
#   projectile_damage  射弹加成         spell_power        法术强度
#   adaptive_force     适应之力         crit_chance        暴击几率
#   crit_damage        暴击伤害         armor              护甲
#   physical_resist    物理抗性         spell_resist       法术抗性
#   physical_penetration      物理穿透  spell_penetration  法术穿透
#   physical_penetration_percent   物穿百分比
#   spell_penetration_percent       法穿百分比
#   damage_reduction   伤害减免
#
# general: 通用奖励（每级累加）
#   属性值 = 配置值 × 当前等级
#   例如 attack_damage: 0.5，玩家 10 级时获得 5 点攻击伤害
#
# specific: 特定等级奖励（达到指定等级时一次性给予）
#   键 = 等级，值 = 该等级解锁的属性
#   例如 10 级时额外获得 5 攻击伤害 + 2% 暴击率
#   注意：specific 奖励不会重复叠加，到达该等级后永久生效
level-rewards:
  general:
    # 每级增加 0.5 攻击伤害
    attack_damage: 0.5
    # 每级增加 1 点生命值
    health: 1.0
  specific:
    10:
      attack_damage: 5.0
      crit_chance: 2.0
      health: 10.0
    20:
      attack_damage: 10.0
      crit_damage: 5.0
      spell_power: 5.0
    30:
      armor: 5.0
      physical_resist: 3.0
      spell_resist: 3.0
    50:
      adaptive_force: 5.0
      damage_reduction: 2.0

# ─────────────────────────────────────
# MythicMobs 经验掉落配置
# ─────────────────────────────────────
# 配置 MythicMobs 怪物死亡时玩家获得的经验值
# 注意：此区域仅在安装了 MythicMobs 插件时生效
monster-exp:
  # 未在下方 mobs 列表中明确配置的怪物，默认给予的经验值
  default: 10

  # 是否根据 MythicMobs 的怪物等级字段自动计算经验值
  # 启用后，若怪物在 MM 配置中设置了 Level 字段
  # 则使用 mob-level-formula 计算经验值，替代 default 值
  use-mob-level: true

  # 基于怪物等级的计算公式（仅 use-mob-level 为 true 时有效）
  # {level} = 怪物在 MythicMobs 中配置的等级
  mob-level-formula: "{level}*5+10"

  # 特定怪物的经验值映射
  # key   = MythicMobs 怪物配置中的内部 ID
  # value = 击杀该怪物获得的经验值
  # 优先级: 此处明确配置 > mob-level-formula 计算 > default
  mobs:
    SkeletonKing: 100
    ZombieLord: 75
    Goblin: 15
    GoblinWarrior: 25
    Dragon: 500

# ─────────────────────────────────────
# 经验获取来源设置
# ─────────────────────────────────────
exp-sources:
  # MythicMobs 怪物击杀经验
  mythicmobs:
    # 是否启用怪物击杀获得经验
    enabled: true
    # 是否仅最后一击的玩家获得经验
    # false 时，以怪物为中心半径 30 格内的所有玩家平分经验
    killer-only: true
    # 是否与队伍成员分享经验（需队伍插件支持，预留）
    share-in-party: false
    # 经验值倍率
    # 所有来源的经验值乘以该倍率，1.0 = 100%
    # 可用于特殊活动时临时调整
    multiplier: 1.0

# ─────────────────────────────────────
# 经验获取提示
# ─────────────────────────────────────
# 玩家获得经验时聊天栏显示的文字
# 支持以下占位符:
#   {player}   - 玩家名
#   {amount}   - 获得的经验值数量
#   {level}    - 当前等级
#   {exp}      - 当前经验值
#   {required} - 升级所需总经验
#   {progress} - 百分比进度（保留1位小数）
# 支持 Minecraft 颜色代码 (&0-&f, &l, &m, &n, &o, &r)
exp-gain:
  # 获得经验时是否在聊天栏显示提示
  enabled: true
  # 提示文字格式
  # 设为空字符串 "" 则不显示
  message: "&a+{amount} &7经验值 (&f{progress}&7)"
  # 获取经验时发送的声音（留空表示不播放）
  # 可用值参考: https://hub.spigotmc.org/javadocs/spigot/org/bukkit/Sound.html
  sound: "ENTITY_EXPERIENCE_ORB_PICKUP"
  # 声音音量 (0.0 ~ 1.0)
  sound-volume: 0.5
  # 声音音调 (0.5 ~ 2.0)
  sound-pitch: 1.0

# ─────────────────────────────────────
# 升级提示配置
# enabled: false 时完全不发送任何升级提示
# ─────────────────────────────────────
# 玩家升级时的提示方式
# mode 可选值:
#   chat      - 聊天栏显示文字消息
#   actionbar - 状态栏（物品栏上方）显示
#   title     - 屏幕中央大标题 + 副标题
level-up:
  # 提示模式: chat / actionbar / title
  mode: title

  # 聊天栏消息（仅 mode = chat 时生效）
  # 可用占位符: {player}, {level}, {old_level}
  chat:
    message: "&b&l✦ {player} &f升级了！ &7({old_level} → &b{level}&7)"

  # ActionBar 消息（仅 mode = actionbar 时生效）
  # 可用占位符: {player}, {level}, {old_level}
  actionbar:
    message: "&b&l✦ 升级！ &7你现在是 &bLv.{level}"

  # 标题消息（仅 mode = title 时生效）
  # 可用占位符: {player}, {level}, {old_level}
  title:
    # 大标题，设为空字符串则不显示大标题
    main: "&b&l✦ 升级！"
    # 副标题，设为空字符串则不显示副标题
    sub: "&7Lv.{level}"
    # 淡入时间（单位：刻，20刻 = 1秒）
    fade-in: 10
    # 停留时间（单位：刻）
    stay: 40
    # 淡出时间（单位：刻）
    fade-out: 10

  # 升级时播放的声音（留空表示不播放）
  sound: "ENTITY_PLAYER_LEVELUP"
  sound-volume: 0.5
  sound-pitch: 1.0

# ─────────────────────────────────────
# MySQL 数据库配置
# ─────────────────────────────────────
database:
  # 数据库地址
  host: localhost
  # 数据库端口
  port: 3306
  # 数据库名
  database: itemcore
  # 表名前缀
  table-prefix: ic_level_
  # 数据库用户名
  username: root
  # 数据库密码
  password: ""
  # 连接池设置
  pool-settings:
    # 最大连接数
    maximum-pool-size: 10
    # 最小空闲连接数
    minimum-idle: 2
    # 连接超时时间（毫秒）
    connection-timeout: 5000

# ─────────────────────────────────────
# PlaceholderAPI 进度条样式
# ─────────────────────────────────────
placeholder:
  progress-bar:
    # 进度条总长度（字符数）
    length: 20
    # 已完成的字符
    done-char: "█"
    # 已完成部分的颜色
    done-color: "&a"
    # 未完成的字符
    not-done-char: "░"
    # 未完成部分的颜色
    not-done-color: "&7"
```

## 5. MySQL 数据存储设计

### 5.1 表结构

```sql
-- ========================================
-- 玩家等级数据表
-- 表名: {table-prefix}player_data
-- 默认: ic_level_player_data
-- ========================================
CREATE TABLE IF NOT EXISTS `ic_level_player_data` (
    `uuid`              VARCHAR(36)     NOT NULL PRIMARY KEY COMMENT '玩家UUID',
    `player_name`       VARCHAR(32)     NOT NULL COMMENT '玩家名（冗余，便于调试）',
    `level`             INT             NOT NULL DEFAULT 1 COMMENT '当前等级',
    `exp`               INT             NOT NULL DEFAULT 0 COMMENT '当前经验值',
    `last_online`       BIGINT          NOT NULL COMMENT '最后在线时间戳（毫秒）',
    `created_at`        TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at`        TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX `idx_player_name` (`player_name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 5.2 数据实体

```java
public class PlayerData {
    private UUID uuid;
    private String playerName;
    private int level;
    private int exp;
    private long lastOnline;
}
```

### 5.3 数据操作

| 操作 | 说明 | SQL |
|------|------|-----|
| 加载 | 玩家加入时异步查询 | `SELECT * FROM ic_level_player_data WHERE uuid = ?` |
| 保存 | 定时 + 退出时 | `INSERT INTO ic_level_player_data ... ON DUPLICATE KEY UPDATE ...` |
| 批量保存 | 定时任务事务批量 | 逐条 INSERT ... ON DUPLICATE KEY UPDATE |

## 7. 属性提供者（与 ItemCore 的集成）

### 7.1 实现方式

```java
public class LevelAttributeProvider implements AttributeProvider {
    @Override
    public AttributeContainer getAttributes(Player player) {
        int level = levelManager.getLevel(player.getUniqueId());
        AttributeContainer container = new AttributeContainer();

        // general: 等级 × 每级属性值
        for (Map.Entry<CustomAttribute, Double> entry : generalRewards.entrySet()) {
            container.addAttribute(entry.getKey(), entry.getValue() * level);
        }

        // specific: 达到等级即生效
        for (Map.Entry<Integer, Map<CustomAttribute, Double>> entry : specificRewards.entrySet()) {
            if (level >= entry.getKey()) {
                for (Map.Entry<CustomAttribute, Double> attrEntry : entry.getValue().entrySet()) {
                    container.addAttribute(attrEntry.getKey(), attrEntry.getValue());
                }
            }
        }

        return container.isEmpty() ? null : container;
    }
}
```

### 7.2 注册

```java
ItemCoreAPI.registerAttributeProvider(new LevelAttributeProvider(levelManager));
```

### 7.3 属性刷新时机

| 时机 | 操作 |
|------|------|
| 玩家升级时 | `ItemCoreAPI.refreshPlayerAttributes(player)` |
| 管理员通过指令调整等级后 | `ItemCoreAPI.refreshPlayerAttributes(player)` |
| 重载配置后 | 刷新所有在线玩家 |
| 服务器启动加载数据后 | 刷新所有在线玩家 |

## 8. 指令设计

### 8.1 指令总览

主指令: `/itemcorelevel`（别名: `/icl`）

| 子指令 | 权限 | 默认 | 说明 |
|--------|------|------|------|
| `icl help` | `itemcorelevel.command.help` | true | 显示插件帮助 |
| `icl info [player]` | `itemcorelevel.command.info` | true | 查看等级信息 |
| `icl add <player> <amount>` | `itemcorelevel.command.add` | op | 增加经验 |
| `icl remove <player> <amount>` | `itemcorelevel.command.remove` | op | 扣除经验 |
| `icl set <player> <level>` | `itemcorelevel.command.set` | op | 设置等级 |
| `icl setexp <player> <amount>` | `itemcorelevel.command.setexp` | op | 设置经验值 |
| `icl reset <player>` | `itemcorelevel.command.reset` | op | 重置玩家数据 |
| `icl reload` | `itemcorelevel.command.reload` | op | 重载配置 |

### 8.2 指令详解

#### help
```
/icl help
```
显示所有可用子指令和简要说明，带颜色和格式化。

#### info
```
/icl info
/icl info <player>
```
- 无参数：查看自己的等级、经验值、本级进度、累计获得的属性加成
- 有参数（需 `itemcorelevel.command.info.other` 权限）：查看指定玩家

示例输出：
```
========= [ 等级信息 ] =========
玩家: &bSteve
等级: &e25
经验: &e1200 &7/ &e2500 &7(&a48.0%&7)
攻击伤害加成: &f+12.5
生命加成: &f+25.0
暴击率加成: &f+2.0%
==============================
```

#### add
```
/icl add <player> <amount>
```
给玩家增加经验值。触发 `PlayerGainExpEvent`（可被其他插件取消）。若经验值达到升级阈值，自动升级并触发 `PlayerLevelUpEvent`。

#### remove
```
/icl remove <player> <amount>
```
扣除玩家经验值。不会导致降级，经验最低为 0。

#### set
```
/icl set <player> <level>
```
直接设置玩家等级。会将当前经验值清零，重新计算属性。

#### setexp
```
/icl setexp <player> <amount>
```
设置玩家的当前经验值。不会自动触发升级检测（与 add 的区别）。

#### reset
```
/icl reset <player>
```
将玩家的等级重置为 start-level，经验重置为 start-exp。调用 `ItemCoreAPI.refreshPlayerAttributes(player)` 重置属性。

#### reload
```
/icl reload
```
重新加载 config.yml，刷新所有在线玩家的属性。

| 参数位置 | 补全内容 |
